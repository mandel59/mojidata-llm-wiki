# L2/26-111

## 要約

`L2/26-111` は、Proposed Draft UTS \#61 "Unicode Set Notation" の draft 4 である。

Unicode Standard、UAX、UTS、data tooling で使われる code points / strings の有限集合表記を、machine-readable な仕様として formalize する。

ICU の UnicodeSet / UTS \#35 に由来する構文を、Unicode specifications の参照可能な set notation として整理し、property queries、set operations、escaping、conformance、style guide を定義する。

PAG report [L2/26-096](utc-l2-26-096.md) は、Unicode Set Notation を draft status に進める項目として扱った。

## 提案内容

### 表記の対象

UTS \#61 は、code points と strings の finite set を property assignments に基づいて表す notation を定義する。UAX \#14、UAX \#29、UAX \#31、UAX \#44、UTS \#51 などの algorithm / data specification で、仕様本文中の集合を機械可読にすることを狙う。

### 構文と操作

Draft は lexical elements、escaped elements、named elements、bracketed elements、property queries、set difference / intersection などの operations を定義する。一般用途 API は syntactically complete かつ minimally consistent な subset を支援し、制限を明示すべきだとする。

### style guide

文書は、Unicode specifications で使う set notation の書き方も定める。POSIX-style property queries を避けること、`\p` を基本にすること、仕様本文では regex match / property comparison / version qualifier を使わないこと、higher-level syntax で comment marker になる `#` は escape することなどを挙げる。

## 後続決定

UTC \#187 agenda [L2/26-092](utc-l2-26-092.md) は UTS \#61 を Properties & Algorithms の public review issue として置く。[L2/26-096](utc-l2-26-096.md) は Unicode Set Notation を draft status に進める項目として記録しており、詳細は [Unicode Set Notation](../topics/unicode-set-notation.md) で追う。

## 論点

### 仕様本文と実装 tooling の共通言語

この proposal は新しい文字の追加ではなく、Unicode specifications が集合をどう書くかを標準化する文書である。既存仕様が UTS \#35 / UTS \#18 / ICU 構文を部分的に参照していた状態を、Unicode Standard 側の UTS として明確にする意味がある。

### higher-level syntax との組み合わせ

Section 6 は、変数置換のような textual replacement ではなく、上位文法に set notation を組み込むことを推奨する。escaping と bidi display の扱いは、Markdown や spec markup で集合式を壊さず表示するための実務上の論点でもある。
