# L2/19-328

## 要約

`L2/19-328` は、`L2/19-282` の `kSpoofingVariant` data が usable ではなく、UTC action item `160-A15` を revised draft data が出るまで hold すべきだと勧告した個人寄書である。author は William T. Nelson、文書本文の日付は 2019-10-02、UTC registry 上の日付は 2019-10-03 である。

文書は、`L2/19-282` が Unihan ad hoc の work product として受け入れられたが、Unihan list members が endorsement した data set ではなく、提出時期も UTC \#160 で public review を確保するには遅かったと指摘する。

| Recommendation M | 今後 UTC は proposal が review されたことをより注意深く確認する。 |
| Data practice | 大量 data を PDF で提出する場合は、text-based format でも data を提供すべきと指摘する。 |

## 論点

### CJK confusables data の review discipline

`L2/19-328` は、`kSpoofingVariant` のような CJK security-related data では、author 以外の expert review、public review のための提出時期、text-based data availability が必要だと主張する。これは、後続の `L2/26-127` が PDF に plain text data を添付したこととも接続する。

### kZVariant との並行問題

文書末尾の note は、`L2/19-281` の `kZVariant` data も `L2/19-282` と同じ issue を持つと述べる。ただし UTC \#160 minutes が `L2/19-281` を参照していないため、本文では `kSpoofingVariant` に絞っている。
