# IRG N2863

## 要約

`IRG N2863` は、GB/T 7589-1987 と GB/T 7590-1987 の連続したrow-cellに対し、`kIRG_GSource` の G3 / G5 values と暫定的な `kGB3` / `kGB5` values が一つ前後へずれている問題を再検証した文書である。

G3では20 ideographs、G5では35 ideographsを原規格の表と照合し、既存値の変更だけでなく、誤ったsourceへの割当ての削除と、欠けている `kGB3` / `kGB5` valuesの追加を提案する。対象問題は `IRG N2302` で既に指摘され、Chinaが `IRG N2376` で変更しないと回答していたが、Unicode 18.0に向けたprovisional propertiesの整備を理由に再提起された。

## 提案内容

### G3

GB/T 7589-1987のrow-cell 16-65から16-83を基準に、連続範囲で一つずれていたG3 referencesを対応する位置へ修正する。U+511Cは `G3-3073` ではなく `GKX-0120.10` とし、対応する `kGB3 1683` を削除する。一方、範囲先頭のU+201B5には欠けていた `kGB3 1665` を追加する。

文書は、U+50B1など一部のG-column glyphが原規格のglyphと異なるのは、前者が対応するunsimplified formであるためとも説明する。このglyph差を、row-cell referenceのずれそのものとは区別している。

### G5

GB/T 7590-1987のrow-cell 67-11から67-25、および73-93から74-18を基準にG5 referencesを修正する。U+8575は `G5-697D` / `kGB5 7393` ではなく `GCESI-00524` として扱い、U+859Eは `GE-4A49` から `G5-6A32` へ移す。

既にG5 sourceを持つ15 ideographsの `kIRG_GSource` valuesを直すとともに、原規格の位置を埋めるため、既存およびsupplementary ideographsへ `kGB5` valuesを追加する。U+7C3BやU+270D2などのglyph差についても、simplified / unsimplified formsの関係を注記している。

### property history

`kGB3` / `kGB5` はUnicode 1.1およびISO/IEC 10646-1:1993以来、対象範囲の多くが変更されていない。`kIRG_GSource` はUnicode 3.0で導入され、Unicode 6.0でsource prefixが `3` / `5` から `G3` / `G5` に変わった後も、該当する位置関係は維持されていた。文書はこの履歴を、単発のdata typoではなく長期間残った系列的なずれとして扱う根拠にしている。

## 後続決定

本文はauthorによるrecommendationをまとめたproposalであり、IRG会合の採択やUnicode dataへの反映を記録したdecision documentではない。PDF内の変更一覧と添付text dataも、提案値の機械可読な提示であって採択記録ではない。

[IRG N2864R](irg-n2864r.md) は後続のChina proposalとして、本書に由来するG3 19 charactersとG5 15 charactersの `kIRG_GSource` reference updatesを、合計1,538 charactersのbulk updateへ取り込んだ。ただし `IRG N2864R` 自体もIRG / WG2での検討を求めるsubmissionであり、その取り込みをもって本書の全提案が採択されたとは判断できない。また本書が扱うprovisional `kGB3` / `kGB5` の追加・削除と、N2864Rが集計するG-source reference updatesは同じ件数ではない。

## 論点

### 連続範囲としての検証

個別code pointだけを見ると既存値にも一見妥当なrow-cellが付いているが、原規格の連続行と突き合わせると、範囲先頭の欠落によって後続値が一つずつずれた構造が見える。提案はその系列全体を同時に修正し、末尾で誤って残った値を別sourceへ戻す。

### normative sourceとprovisional property

`kIRG_GSource` はnormative propertyである一方、`kGB3` / `kGB5` はprovisional propertiesである。本書は同じ原規格を参照する両者をまとめて整合させるが、後続文書のG-source件数だけではprovisional propertiesへの処置まで確認できない。
