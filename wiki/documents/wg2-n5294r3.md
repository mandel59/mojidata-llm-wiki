# WG2 N5294R3

## 要約

`WG2 N5294R3` / `L2/25-049R` は、Michel Suignard による "Considerations concerning the Small Seal encoding initiative" である。文書は [WG2 N5209](wg2-n5209.md) を置き換え、2024 年以降の contribution を踏まえて、Small Seal Script の repertoire と corresponding database を提示する。

著者は、この文書時点で Small Seal repertoire は mature であり、少数の variants の disunification を最終確認すれば encoding ready に近いと見る。対象 repertoire は 11,140 code points で、THX / CCZ / QJZ / DYC の source sets と、modern CJK equivalents、radical data を持つ database model が示される。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| source sets | THX / CCZ は Daxu Ben 系、QJZ は Xiaoxu Ben 系、DYC は Duan Yucai annotated text 系として扱う。 |
| repertoire | THX 11,108 entries から合意済み variant 17 件を unified にし、QJZ / DYC unique entries の整理と追加 disunification を経て 11,140 code points とする。 |
| chart model | multi-column chart で source references、Small Seal glyph、modern CJK equivalent、radical number を示す。 |
| database fields | `kSEAL_THXSrc`、`kSEAL_CCZSrc`、`kSEAL_DYCSrc`、`kSEAL_QJZSrc`、`kSEAL_MCJK`、`kSEAL_Rad` を定義する。 |
| variant review | Cook mapping 由来の 42 variant sets のうち、17 sets は当初合意済み、8 sets は後続で合意済み、残る 17 sets は未統合として review 対象にする。 |

## 後続決定

`WG2 N5294R3` は consideration / proposal input であり、会議決定ではない。[WG2 N5312](wg2-n5312.md) は N5294R2 / N5294R3 に対する TCA and China feedback として、17 variant sets、19 separate-code candidates、mapping errors、THX glyph corrections を返した。[WG2 N5306](wg2-n5306.md) は、この model を受けて最終 proposal 前の unification / disunification issue を整理した。

WG2 \#72 での future edition candidate への接続は、[WG2 M72.13 Small Seal future edition candidate](../events/wg2-m72-13-small-seal-future-candidate.md) に集約する。

## 論点

### N5209 からの前進

[WG2 N5209](wg2-n5209.md) は source mapping comparison と database considerations の段階だった。`WG2 N5294R3` は、そこから full glyph coverage と draft repertoire / database に進み、11,140 code points を concrete encoding candidate として扱う段階へ進めた。

### database field と character identity

文書は `kSEAL_*Src` values を source identity の中核として扱い、`kSEAL_MCJK` と `kSEAL_Rad` を検索・索引・chart presentation に使う。後続の [WG2 N5346](wg2-n5346.md) で大量の modern CJK correction が入ることからも、modern equivalent は useful data であって source identity そのものではない。
