# IRG N2840

## 要約

`IRG N2840` は、同じように見えるglyphが複数code pointsに現れる問題を、4組のG-source ideographsについて調べた個人 contribution である。新しい辞書evidenceを示し、片方のrepresentative glyphを別の字形へ改めることと、対応するsource referencesの修正をChina、IRG、WG2に求める。

文書はduplicate encodingがsearch、sorting、comparison、frequency count、tokenization、copy-and-paste、diff等へ及ぼす影響を動機として挙げる。本文の4組とは別に、同様に見える組をappendixへ列挙するが、それらを一括してdisunifyまたは統合する提案ではない。

これはglyphとreferenceの修正案を提示するproposalであり、どの案を採択したかを記録したdecision documentではない。

## 提案内容

文書が扱う4組と問題の類型は次のとおりである。glyph画像そのものはPDF中の表とevidenceで確認する必要がある。

| 対象 | 問題の類型 | 提案の焦点 |
| --- | --- | --- |
| U+34A8 / U+20457 | 二つのcode pointsのG-source glyphが重なる | `GHZR-10280.06` / `GHZR-10281.02` の新しいevidenceにより、どちらのglyphとreferenceを動かすべきかを問い直す。 |
| U+821D / U+246C9 | URO characterとExtension B characterのglyphが重なる | U+821D側の `GHC-010797.02` と、U+246C9側の `GHZ-0854.55` を対照し、後者を別字形として表す案を示す。 |
| U+204F2 / U+23515 | 同一に見えるExtension B glyphとreferenceの割当て | 連続するevidence `GHZR-10237.05` / `GHZR-10237.06` に従い、両code pointsを異なるglyph・referenceへ対応させる。 |
| U+249BC / U+249E9 | 二つのExtension B code pointsのglyphが重なる | `GHZR-21200.09` 等のevidenceから、片方のrepresentative glyphを修正して区別する。 |

### glyph と source reference

本提案では、representative glyphの変更とsource referenceの変更が密接に関係するが、同じ操作ではない。glyphはcode chartに示す地域別代表字形であり、referenceはその字形を裏付けるsource material中の位置を指す。新しいreferenceを発見しても、既存code pointのglyphを必ず変更すべきとは限らず、安定性や元のevidenceとの一致を個別に評価する必要がある。

また、二つのglyphが重なることは、code points自体を統合できることを意味しない。文書の具体的要求は、既存の符号位置を維持したままG-source glyphとreferenceの対応を正すことである。

## 後続決定

`IRG N2840` 自身は採択結果を含まない。Chinaの後続feedback `IRG N2874R` は4組を再検討し、U+34A8とU+821Dは元のevidenceおよび安定性を理由に変更せず、対になるU+20457とU+246C9を修正する方向を示した。U+204F2 / U+23515とU+249BC / U+249E9についても、glyphとreferenceの具体的な割当てを整理した。これは本提案へのresponseであり、その内容を `IRG N2840` の決定として遡及させない。

U+246C9とU+249E9のreference updatesは [IRG N2864R](irg-n2864r.md) の1,538 G-source reference updatesにも含まれる。一方、同文書は両文字のglyph updatesを `IRG N2874R` で扱うとしており、bulk reference updateとrepresentative glyph decisionの境界を明示している。

その後のIRG recommendationやUTCでの採択は別のdecision recordsで確認すべきであり、本ページでは提案時点と後続feedbackの境界を維持する。

## 論点

### duplicate glyphへの対処

同じglyphを異なるcode pointsが持つ状態はNLPや検索に実務上の負担を生むが、既存文字の安定性も重要である。このため、単に一方を変えるのではなく、各source evidenceに照らしてどのcode pointにどの字形を残すかを判断する必要がある。

### UROと拡張面で異なる安定性

U+34A8やU+821Dのような既存利用の広い文字と、対になるExtension B charactersを同じ条件で変更できるとは限らない。後続feedbackが既存URO glyphを維持する方向を取ったことは、この安定性の差を示す。

### 個別glyph issueとbulk update

`IRG N2840` は4組の形態的な問題から出発するのに対し、[IRG N2864R](irg-n2864r.md) は多数のreference valuesとsource descriptionsをまとめて更新する。共通するcode pointsがあっても、前者のglyph分析と後者のdata updateを同一のdecisionとして扱わないことが重要である。
