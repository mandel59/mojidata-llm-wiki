# IRG N2838

## 要約

`IRG N2838` は、`IRG N2835` が提案した GHC / GCY / GCH source reference changes を、既存の syntax、辞書実物、過去の submission data と照合した個人 contribution である。

主な指摘は、GHC の末尾 swift digit、GCH の《辞海》edition description、GCY の page / column / 同一欄内連番の区別である。加えて GGFZ reference 1件、China の regional convention に合わない representative glyphs、Extension F で source suffix の欠落により postponed された文字を扱う。

文書は修正案と evidence を提示する feedback であり、変更の採択を記録した decision document ではない。

## フィードバック内容

### GHC-source

`IRG N2442` の `GHC-xxyyyyz` syntax では、末尾 `z` は同一page内の複数文字を0から数える swift digit である。しかし `IRG N2835` では、page内の最初または唯一の文字にも恣意的に `1` が付く例があるとして、多数の値を `...1` から `...0` へ修正する一覧を示す。代表例は U+20666 の `GHC-0104241` から `GHC-0104240` への変更である。

また U+2AD81 と U+2B46B の G-source representative glyphs が China の regional convention と一致しないとして、GB 18030—2022 のglyphに合わせるよう求める。

### GGFZ-source

U+235CB の値が placeholder の `GGFZ-nnnnnn` のままであることを指摘し、《通用规范汉字字典》page 67 の evidence に基づく `GGFZ-006700` を提案する。

### GCH-source

`IRG N2835` は source を《辞海》1999年版縮印本と説明する一方、提示したpage numbersは普及本に対応すると指摘する。U+2B806について、`IRG N1532R` の縮印本 evidenceによる `GCH-1606.02` と、提案値 `GCH-3772.02` が参照する普及本を対比し、descriptionを「1999年版普及本」へ直すか、valuesを縮印本に合わせるよう求める。

さらに U+2AC7E、U+2AD51、U+2ADE4、U+2B077、U+2B416 の representative glyphs を GB 18030—2022 に合わせるよう提案する。

### GCY-source

U+2A92F の従来値と Extension F の値は同じ形式に見えても意味が統一されていないと分析する。`IRG N1983` の値は page と head character positionを表す一方、`IRG N1886` の Extension F data は page number と optional letter に由来し、後の変換で意味が曖昧になった。

そこで《辞源》自身と `IRG N1886` appendix の表記に合わせ、`GCY-xxxx.yz` を page、column、同一column内の swift digit と定義する案を示す。個別の修正表に加え、U+27D2E、U+28201、U+284CF、U+2C7B1 には新版《辞源》から新しい evidence を提示する。

Extension F では、original source values の suffix が `IRG N1921` 作成時に欠落し、重複sourceとしてpostponeされた文字が残った経緯を示す。そのうち5文字について GCY horizontal extension をChinaが提出し、未符号化の1文字は将来の IRG Working Set に提出するよう勧める。

## 後続決定

4日後の `IRG N2835R` では、確認できる範囲で `IRG N2838` の主要な reference 修正が反映された。GHC values は page内の先頭を `0` とする形へ改められ、U+235CB は `GGFZ-006700` となり、GCH source description は《辞海》1999年版普及本へ変更された。GCY section は `IRG N2838` を詳細根拠として明記し、`GCY-xxxx.yz` に沿う提案値を採用している。ただし `IRG N2835R` 自体も proposal であり、IRG の採択記録ではない。

[IRG N2868](irg-n2868.md) はその後、GCH / GCY の値を original submission 時の edition と printing に遡って再検討し、`IRG N2838` の修正を最終結論とは扱っていない。[IRG N2864R](irg-n2864r.md) は `IRG N2835R` と `IRG N2838` 等の evidence を取り込み、GCH / GHC / GCY を含む1,538 charactersの bulk update proposalへ接続した。これらの後続proposalとfeedbackを、`IRG N2838` 自身の決定としては扱わない。

## 論点

### reference syntax と再現可能性

source reference は単なるpage locatorではなく、volume、page、column、同一位置の複数文字を一意に再現できる必要がある。既存形式を機械的に揃えるだけでは、由来の異なるsuffixの意味を混同する危険がある。

### bibliographic description と実際のpage

GCHの例は、辞書名と年版だけでなく、縮印本・普及本・printingの区別がreference valueの意味を左右することを示す。この問題は後続の `IRG N2868` でoriginal submissionの版・刷を特定する課題へ発展した。

### source data と representative glyph

文書は source reference correction と representative glyph normalization を別の問題として提示する。reference値の修正が、そのままglyph変更の承認を意味するわけではない。
