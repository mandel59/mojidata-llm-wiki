# WG2 N5209

## 要約

`WG2 N5209` は、Michel Suignard が Small Seal encoding initiative の進め方を整理した individual contribution / project editor report である。TCA / China の THX-based proposal と Richard Cook の [L2/22-279](utc-l2-22-279.md) source mapping data を比較し、multi-source database と variant handling の作業方針を示す。

文書は、X / THX を主 index としつつ、QJZ と DYC を含む 3-source comparison を扱う。540 radicals は各 source set に含まれるが、radicals を別途 encode する consensus はないと整理する。

## 報告内容

| 項目 | 内容 |
| --- | --- |
| compared sources | X / THX、K / QJZ、D / DYC。 |
| TCA / China set | 2022 年時点の THX-based set は 17 entries を unification で除外し、total 11,091 と説明される。 |
| Cook mapping | [L2/22-279](utc-l2-22-279.md) は 3 sources を対応づけるが、glyphic / graphemic evidence は十分でないと評価される。 |
| variant sets | 共通の 17 pairs、K / D only の 8 pairs、Cook 側の 41 variant sets などを比較対象にする。 |
| working table | 11,163 entries を flat table として統合し、encode しない entries に flag を付ける形を示す。 |
| review focus | Cook data の `+` / `-` qualifiers が付く約 500 entries を、追加 disunification review の焦点にできるとする。 |

文書は、TCA / China と Cook の差分を単にどちらかへ寄せるのではなく、source evidence と variant sets を表形式で突き合わせる方向を提案する。Radical \#2 の regular form と radical-form variant を両方 encode するかは未解決 issue として残している。

## 後続決定

この文書自体は報告・検討文書であり、決定文書ではない。後続では [WG2 N5230](wg2-n5230.md) の TCA feedback、[WG2 N5273](wg2-n5273.md) の TCA / China response、2025 年の mature repertoire、[WG2 N5344R2](wg2-n5344r2.md) の revised proposal へ進み、最終的に THX / CCZ / QJZ / DYC の four-column source model と UAX \#60 / SealSources.txt へ接続する database が整理された。

## 論点

### Variant set の扱い

N5209 は、TCA / China が採用した 17 variant pairs と Cook data の 33 X-source variant pairs を比較し、33 pairs が 17 pairs を含む一方で、共通 17 pairs のうち 6 pairs は preferred variant の向きが逆になると指摘する。著者は例示された X914 / X8016 では TCA / China 側を好むと述べ、mechanical merge ではなく evidence review が必要だとする。

### Multi-source database

文書は、各 proposed code point に source references、glyph、modern CJK equivalents、radical modern form、alternate radical、radical number を持たせる database 形式を示す。ここで述べられた `kSEAL_THXSrc`、`kSEAL_MCJK`、`kSEAL_Rad` などの property idea は、後続の [UAX \#60 Data for Large East Asian Scripts](../topics/uax60-large-east-asian-scripts.md) と [WG2 N5344R2](wg2-n5344r2.md) に接続する。

### Allocation

N5209 時点では proposed block start として `U+38000` が使われていた。後続の revised proposal では allocation が `3D000..3FC3F` に移り、WG2 \#73 の CD.4 disposition でもこの範囲が前提になった。
