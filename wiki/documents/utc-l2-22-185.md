# L2/22-185

## 要約

`L2/22-185` は、UAX \#45 の status values を一文字 code から説明的な multi-letter values へ変更する proposal である。Unicode 15.0 で一部 status values が multi-letter に変更された後、`N`, `V`, `W`, `X` が残っており、意味が直感的でなく将来 ambiguity を生む可能性があるとする。

## 提案内容

文書は、残る一文字 status values を次のように置き換えることを勧告する。

| 旧 status | 新 status |
| --- | --- |
| `N` | `FutureWS` |
| `V` | `Variant` |
| `W` | `Rejected` |
| `X` | `NoAction` |

この proposal は UAX \#45 data の readability を上げ、status の意味を data consumer が理解しやすくすることを目的にしている。

## 後続決定

[L2/25-199](utc-l2-25-199.md) の UAX \#45 Revision 31 proposed update では、`FutureWS`、`Variant`、`Rejected`、`NoAction` が status table に含まれている。`FutureWS` は、UTC が将来の IRG working set submission に含める予定の ideograph を示す status として使われる。

## 論点

status value の命名は、U-source records の machine readability と human reviewability の両方に効く。特に `FutureWS` は、UAX \#45 additions が単なる backlog ではなく、IRG working set pipeline へ渡す候補であることを明示する。
