# WG2 N5344R2

## 要約

`WG2 N5344R2` は Small Seal Script を UCS に符号化する revised proposal である。対象は `Shuowen Jiezi` に基づく 11,328 characters で、主 source は `THX`、補助 source として `CCZ`、`QJZ`、`DYC` を含む。

提案は four-column code chart と database を持ち、source references、modern CJK equivalents、radical data、Unicode properties、collation、proposal summary form をまとめている。過去文書として [WG2 N4688](wg2-n4688.md)、[WG2 N5105](wg2-n5105.md)、`N5211`、[L2/22-279](utc-l2-22-279.md)、[WG2 N5209](wg2-n5209.md)、`N5306`、[WG2 N5307R](wg2-n5307r.md)、[WG2 N5317R3](wg2-n5317r3.md)、`N5346` などを取り込んだ集約版である。

## 提案内容

| 項目 | 内容 |
| --- | --- |
| repertoire | `Shuowen Jiezi` に基づく 11,328 Small Seal characters。 |
| allocation | `3D000..3FC3F` の block を提案し、block は 11,328 code points で埋まる。 |
| source model | THX を primary ordering / source reference とし、CCZ、QJZ、DYC を four-column chart に並べる。 |
| data | 各 code point に最大 4 source references、modern CJK equivalent、1 つまたは 2 つの radical numbers を持たせる。 |
| data properties | `kSEAL_THXSrc`、`kSEAL_CCZSrc`、`kSEAL_QJZSrc`、`kSEAL_DYCSrc`、`kSEAL_MCJK`、`kSEAL_Rad`。 |
| collation | Radical group order で encode されるため、code point order を collation order として使えると説明する。 |

この文書自体は proposal であり、決定文書ではない。後続の [WG2 Meeting \#73](../meetings/wg2/wg2-meeting-73.md) では、CD.4 disposition の significant changes として Small Seal character names と 26 characters の property correction が記録された。

## 論点

### 独立 script としての位置づけ

文書は、Small Seal が modern Hanzi の calligraphy typeface ではなく、Oracle Bone script、Bronze inscriptions、Warring States scripts と同様に ancient Hanzi development の一段階を表す文字集合だと説明する。modern CJK と一対一対応しない文字や独自構造が多いため、UCS では CJK Unified Ideographs の glyph variation ではなく独立 script として扱うべきだとする。

### Four-column source model

THX は Daxu Ben の主要 version として repertoire と ordering の基準になる。CCZ も Daxu Ben 系、QJZ は Xiaoxu Ben、DYC は Duan Yucai の annotation tradition を代表する。THX に存在しないが他 source にある 237 supplemental glyphs は `TH-Yddd` syntax で THX column に置かれ、THX ordering へ接続される。

### Unification / disunification criteria

同じ writing form の glyphs は form と explanation の整合を見て unified / disunified される。この判断基準は [WG2 N5307R](wg2-n5307r.md) が整理している。一方、Zhengzhuan、Guwen、Zhouwen のような written forms は、似た形でも Shuowen system の integrity を保つため別 characters として扱われる。

### Small Seal database

`kSEAL_*Src` values は character identity を担う source references であり、UAX \#60 / `SealSources.txt` へ接続する。modern CJK equivalent は検索・索引用に有用だが、文書は work in progress として扱い、normative identity は sources が担うと整理する。

### Script / block name

N5344R2 は block allocation として `3D000..3FC3F` を提案し、script property と character names には ISO 15924 との整合から `Seal` を使う案を示していた。一方、WG2 \#73 では character names を `SMALL SEAL CHARACTER-XXXXX` に変更する方向になった。

### Modern CJK equivalents

modern CJK equivalent は検索・索引用に重要だが、normative identity は source reference が担う。N5344R2 は modern CJK equivalent が今後更新され得る work in progress であることを明示している。後続の [WG2 N5355](wg2-n5355.md) は、`kSEAL_MCJK` を `zhèngzhuàn`（正篆）/ `chóngwén`（重文）の関係で説明し直す。

## 後続決定

[WG2 N5369](wg2-n5369.md) の CD.4 disposition は、N5344R2 の proposed encoding を 7th edition DIS へ進める過程で、names を `SMALL SEAL CHARACTER-XXXXX` に変更し、`kSEAL_Rad` 6 characters、`kSEAL_MCJK` 20 characters、U+3F80E glyph を修正した。[WG2 N5354](wg2-n5354.md) の M73.01 / M73.04 が、この disposition を SC2 / DIS progression に接続する。
