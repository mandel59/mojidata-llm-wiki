# L2/26-106

## 要約

`L2/26-106` は、Proposed Update UAX \#53 Revision 12 "Unicode Arabic Mark Rendering"、Unicode 18.0.0 draft 1 である。

文書は Arabic Mark Transient Reordering Algorithm (AMTRA) を定義し、Arabic combining marks を含む sequences が、canonically equivalent な入力順でも同じ表示結果になるよう rendering pipeline 内で一時的に並べ替える方法を示す。

AMTRA は normalization form でも stored text の変更規則でもない。実装が display や編集操作のために使う transient ordering であり、原文 text や interchange data を書き換えないことが前提である。

## 提案内容

### AMTRA の目的

Arabic script では、canonical combining class の値と利用者が期待する mark stacking order が単純に一致しない場合がある。AMTRA は、まず NFD 相当の ordering を得たうえで、shadda、Modifier Combining Marks (MCM)、ccc=230 / ccc=220 の marks を base に近づける調整を行う。

これにより、hamza above / below、small high seen、small high yeh、small high / low noon、large round dot above / below などを含む sequences で、内側から外側へ積まれる表示順を安定させる。

### Modifier Combining Marks

MCM は、Arabic marks のうち base character を直接修飾するものとして扱う group である。文書は、既存文字を後から MCM に加えると rendering behavior が変わるため、MCM set は安定させ、追加は原則として新規符号化時に行うと説明する。

### CGJ による例外指定

default display order が特定の orthography と合わない場合は、U+034F COMBINING GRAPHEME JOINER (CGJ) を使って AMTRA の reordering を抑制する。文書の examples は、hamza / damma、kasra / hamza below、small high seen、small high yeh、small high noon、small low noon with fatha / damma などで CGJ の使い方を示す。

### dotted circle と編集操作

Arabic script では、lesser-known orthography や Quranic marks の組み合わせが rendering engine から invalid に見える場合があるため、dotted circle insertion は rendering engine より高い層で扱うべきだと説明する。AMTRA は stored text には適用しないが、backspace 処理で外側の mark を判断するような editing behavior には参考になり得る。

## 論点

### normalization ではなく rendering algorithm

AMTRA は canonical equivalence を保った表示結果を得るための rendering-side algorithm であり、Normalization Form を追加したり、stored text を変更したりするものではない。この点は、文字列比較、検索、入力 method、font shaping の責任分担を考える上で重要である。

### property data と script behavior の境界

MCM set、canonical combining class、Diacritic property、CGJ override は、UCD property と shaping behavior の境界にある。UAX \#53 は Arabic-specific な rendering guidance だが、PAG report と合わせると Unicode 18.0 の properties / algorithms update の一部として読む必要がある。
