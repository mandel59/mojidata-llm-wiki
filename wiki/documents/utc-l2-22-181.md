# L2/22-181

## 要約

`L2/22-181` は、provisional Unihan Database property `kJapanese` を追加する提案である。既存の `kJapaneseKun` / `kJapaneseOn` が modified Hepburn transliteration の uppercase Latin values を持つのに対し、`kJapanese` は Moji Jōhō Kiban database 由来の Japanese readings を hiragana / katakana で記録する。

提案 property は 51,582 CJK Unified Ideographs を cover するとされ、initial data は Moji Jōhō Kiban version 006.01 の readings に限定する方針を示す。UAX \#38 の field description では、hiragana readings は generally Kun-yomi、katakana readings は generally On-yomi と説明される。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| 提案 property | `kJapanese`。 |
| status | Provisional。 |
| syntax | Hiragana / Katakana / voiced marks / prolonged sound mark を含む kana string の space-delimited values。 |
| source data | Moji Jōhō Kiban database version 006.01。 |

## 論点

### kana readings と on/kun の境界

提案時の description は、hiragana が generally Kun-yomi、katakana が generally On-yomi とする。後続の `L2/25-213` は、この説明に例外があり、katakana 表記の kun'yomi が存在すると指摘した。したがって `kJapanese` は、kana 表記そのものを data として保存する property であり、kana script style から on/kun を機械的に決める property ではない。

### 既存 properties との関係

文書は、将来的に provisional `kJapaneseKun` / `kJapaneseOn` を除去する構想にも触れる。ただし当面は両者を残し、overlap する readings や `kJapaneseKun` / `kJapaneseOn` だけを持つ 141 ideographs の比較を可能にすると説明する。
