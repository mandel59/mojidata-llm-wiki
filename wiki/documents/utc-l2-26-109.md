# L2/26-109

## 要約

`L2/26-109` は、Proposed Update UTS \#10 Revision 54 "Unicode Collation Algorithm"、Unicode 18.0.0 draft 3 である。

Revision 54 は Unicode 18.0.0 向けの UCA / DUCET update として、Tibetan contractions を DUCET well-formedness の観点から説明し、noncharacters U+FFFE / U+FFFF の reserved weights と special handling を明確化する。

文書はまた、"collation element" を "CE" と略す note を UTS10-D2 に追加する。

## 提案内容

### Tibetan contractions と DUCET well-formedness

Revision 54 は、DUCET の well-formedness condition を保つために Tibetan contraction mappings が必要になる理由を説明する。単に 2 個の contractions を追加すると Tibetan の default sort order を乱すため、sort order を保つための 8 個を加え、合計 10 個の Tibetan contractions を扱う。

対象には、deprecated Tibetan characters U+0F77 TIBETAN VOWEL SIGN VOCALIC RR と U+0F79 TIBETAN VOWEL SIGN VOCALIC LL を含む sequences が含まれる。文書は、通常の Tibetan text ではこれらの deprecated characters は多く使われないとも説明する。

### U+FFFE / U+FFFF の special weights

Revision 54 は Reserved Weights の subsection を分け、U+FFFE と U+FFFF の扱いを明確化する。

U+FFFE は fixed lowest primary weight `0200` に map され、alternate handling / variable weighting で ignorable にならないこと、identical level でも minimal unique weight を出すことが必要とされる。U+FFFF は fixed highest primary weight `FFFF` に map され、文字列範囲指定で上限 sentinel のように使える。

### CE abbreviation

UTS10-D2 に、collation element を "CE" と略す note が追加される。これは仕様本文と DUCET / CLDR data の説明を読む際の用語整理である。

## 後続決定

[L2/26-096](utc-l2-26-096.md) は UTC \#187 の PAG report として、UTS \#10 / DUCET / CLDR alignment の一部に Tibetan contractions と U+FFFE / U+FFFF weights を挙げる。

## 論点

### DUCET と CLDR の同期

UCA の default table は DUCET として Unicode release に結びつく一方、実装では CLDR collation data と合わせて使われることが多い。U+FFFE / U+FFFF の special handling は CLDR 側の既存 behavior とも揃える必要がある。

### noncharacters の collation behavior

U+FFFE / U+FFFF は noncharacters だが、collation では sort key generation や range specification の境界として意味を持つ。noncharacter であることと、algorithm 上の special weight を持つことを混同しないように読む必要がある。
