# Proposal to enhance the syntax of the RSIndex.txt data file

## 要約

`L2/26-134` は、`RSIndex.txt` の syntax を拡張する提案である。
`RSIndex.txt` は radical-stroke value と、その `kRSUnicode` value を持つ Han ideographs の ordered list を対応させる data file である。
現状の order は traditional radical、Chinese simplified radical、non-Chinese simplified radical、second non-Chinese simplified radical の順だが、file 内では group boundary が機械可読に示されていない。
提案は、second field に one / two / three `|` separators を挿入し、4 つの groups を明示する。
実装者にとっては、radical-stroke index の生成、diff、UI 表示で group boundary を取り出しやすくする変更である。

## 提案内容

| 項目 | 内容 |
| --- | --- |
| 対象 file | `RSIndex.txt`。 |
| 現行構造 | first field は radical-stroke value、second field は ordered Han ideographs の scalar values。 |
| 追加 syntax | second field に `|`、`||`、`|||` を挿入し、traditional / Chinese simplified / non-Chinese simplified / second non-Chinese simplified radical groups を分ける。 |
| 対応 property | `kRSUnicode` では simplified radical の種類を apostrophe、double apostrophe、triple apostrophe で表す。 |
| header update | file header に tab-delimited fields と separator の意味を説明する text を追加する。 |

例として、radical #212 residual strokes 5 の行では、従来の code point list に group separators を挿入し、`U+9F92 U+2A699 U+3133E | U+31342 || U+25269 U+2AF5E U+323AC ||| U+32C47` のように group boundary を示す。文書は、実 ideograph を見せる comment line には `|` を入れない方針を示す。

## 後続決定

この proposal は 2026-05-21 の UTC registry 文書であり、UTC #187 の `L2/26-099` より後に提出されている。現時点の wiki では、後続の UTC minutes / action item による処理は未確認である。[Unicode 18.0 Change Sources](../topics/unicode-18-change-sources.md) では、Unicode 18.0 beta data と UTC #188 以降の確認対象として扱う。

## 論点

### syntax change と互換性

`RSIndex.txt` を whitespace-separated code point list として単純に読む実装は、separator token を扱う必要がある。逆に、radical group boundary を推定していた実装は、`|` tokens を使って明示的に group を得られる。

### IRG indexing rules との接続

radical-stroke index は [IRG Indexing Rules](../topics/irg-indexing-rules.md) の radical assignment / stroke count と接続する。ただし `L2/26-134` の主眼は IRG rule の変更ではなく、UAX #38 周辺で配布される data file の machine-readable syntax 変更である。
