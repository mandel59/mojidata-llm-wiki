# IRG N2656

## 要約

`IRG N2656` は、`Unihan_IRGSources.txt` の `kIRG_GSource` に残る重複referenceを一意化する提案である。G4K、GBK、GCH、GCY、GHCに属する計955 charactersについて、既存の短いreferenceへ対象code pointを表す `-Uxxxxx` を付加する。

狙いはsource referenceからcode pointへのdirect lookupを可能にすることであり、各辞書内の実在位置を新たに確定することではない。提案はUAX \#38のsyntax変更とdata更新を伴うが、Chinaの同意とIRGでの検討を必要とするproposalであり、本文自体は採択記録ではない。

## 提案内容

### 一意性原則と問題例

IRG source referenceは原則として一意であり、同じ値を複数code pointsへ割り当てないという前提に立つ。ところがExtension Bには、G4K 3、GBK 86、GCH 247、GCY 66、GHC 553の計955 charactersで重複がある。短縮されたreference表記自体はUAX \#38のregular expressionsが認める有効な形式だが、一意でないためreference valueをkeyとするdirect lookupができず、検索によって候補を列挙する必要がある。

文書は各重複値へ `-U` と5桁のhexadecimal code pointを付加する。たとえば既存の短いsource valueを維持しながらcode point部分で衝突を避け、通常の辞書位置型referenceとも識別できる形式にする。このためG4K、GBK、GCH、GCY、GHCに対応するUAX \#38のsyntaxも、従来形式と `Uxxxxx` 形式の双方を受理するよう変更する。

`GZ-4321207` の2件も同種の重複だったが、IRGの確認後にUTCがすでに修正していたため、本提案の変更対象から除外されている。

### 暫定値と望ましい代替案

文書自身が「最善の代替案」とするのは、IRG expertsが辞書を調査し、該当文字の実際の掲載位置を通常形式のsource referenceとして提示する方法である。`-Uxxxxx` はその言語学的・書誌的調査を完了した値ではなく、現存dataを機械的に一意化する解決策である。またG-source referenceの変更にはChinaの同意が必要だと明記する。

## 後続決定

[IRG N2689](irg-n2689.md) は本提案を参照し、GCH 624、GCY 192、GHC 601 charactersを含む1,685 G-source referencesの更新案へcode point型referenceを取り込んだ。ただし同文書もChinaからのproposalであり、`IRG N2656`の一意化方式が最終決定されたことを意味しない。

[IRG N2864R](irg-n2864r.md) は後続のevidence reviewを統合し、GCH / GCY / GHCについてcode point型の暫定referenceから、辞書上の位置を示すreferenceへ戻す方向を提案した。したがって本提案の影響は、大規模data整理を始めるための一意性要件と中間的な表現方法を示した点にあり、後続議論の最終的なreference valuesを定めた点にはない。

## 論点

### identifierとしての一意性とsource evidence

本提案は、source referenceをidentifierとして利用するための一意性を優先する。一方、code point由来のsuffixはsource dictionary内の位置を表さないため、source evidenceの精度を高める作業とは区別される。後続文書で実在のpublishing sourceと掲載位置が確認された場合、本提案の暫定値を置換し得る。

### data修正と採択の境界

本文が提示するのはUAX \#38 regular expressionsと`Unihan_IRGSources.txt` entriesのproposed modificationsである。IRG、UTC、WG2による承認や、Unicode dataへの反映完了を記録するminutes / recommendationではない。このため955件を採択済み変更として扱わず、後続のdecision文書または公開data versionで個別に確認する必要がある。
