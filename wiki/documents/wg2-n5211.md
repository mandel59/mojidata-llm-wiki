# WG2 N5211

## 要約

`WG2 N5211` は、Suzuki Toshiya と Richard Cook による `14-Column Seal Script Glyph Comparison Chart` の documentation である。2017-12-06 に WG2 Seal Script Ad Hoc 向けに作られた 631-page chart を、2023-04-03 に archival reference として WG2 register に載せた文書である。

documentation は、chart の 14 columns を `Shuowen Jiezi` versions / fonts に対応づけ、X / D / K の 3 source classes を説明する。後続の [WG2 N5209](wg2-n5209.md)、[WG2 N5230](wg2-n5230.md)、[WG2 N5273](wg2-n5273.md)、[WG2 N5294R3](wg2-n5294r3.md) は、この chart と [L2/22-279](utc-l2-22-279.md) の source mapping を前提に、THX / QJZ / DYC などの差分を検討した。

## データ内容

| 区分 | 内容 |
| --- | --- |
| chart | 14-column Seal Script glyph comparison chart。documentation は chart page 1 / 631 を注釈し、circled column numbers と color coding を説明する。 |
| source classes | X は Xu Xuan / Da Xu ben、D は Duan Yucai / `Shuowen Jiezi Zhu`、K は Xu Kai / Xiao Xu ben を表す。 |
| row model | glyph の下に serial number / Glyph ID を置く。source に該当 form がない column は空欄になる。 |
| variant example | `D00007` / `00007` / `00008` など、`二`、`丄`、`上` の structurally different だが synonymous な forms を例に、unification rules の材料になると説明する。 |
| archival data | register の related link には 2017-12-06 chart data PDF があるが、容量は 420MB である。 |

## Source classes and columns

| Class | Columns | 内容 |
| --- | --- | --- |
| X | ① 王, ② 孫, ③ 陳, ④ N4688, ⑤ TCA, ⑥ 毛4, ⑦ 毛5, ⑧ 華東, ⑨ 北師, ⑩ 逢甲 | Da Xu ben / Xu Xuan 系。Song ben、Pingjinguan、CCZ、THX2、TCA font、Jiguge 系、大学・民間 font を含む。 |
| D | ⑪ 段 | Duan Yucai `Shuowen Jiezi Zhu` / DYC。Kawabata Taichi の Kanji Database Project 由来画像を使う。 |
| K | ⑫ 述, ⑬ 汪, ⑭ 祁 | Xiao Xu ben / Xu Kai 系。Shugutang、Wang Qishu、Qi Junzao editions を含む。 |

documentation は、各 column の source material と画像・font の由来を詳述する。たとえば ④ N4688 / THX2 column は [WG2 N4688](wg2-n4688.md) の Tenghuaxie images から取り、⑤ TCA column も N4688 の proposed representative glyph column から取るが、後続会合で変更が加わったため latest proposal とは同一ではないとする。

## 後続決定

`WG2 N5211` 自体は data / archival reference であり、会議決定ではない。[WG2 N5209](wg2-n5209.md) は TCA / China proposal と Cook source mapping を比較し、N5211 のような multi-source chart を前提に database 化の方向を示した。[WG2 N5273](wg2-n5273.md) は、N5211 と N5230 を背景に N5209 の 55 entries を再検討した。

後続の [WG2 N5344R2](wg2-n5344r2.md) では、N5211 の chart work と [L2/22-279](utc-l2-22-279.md) の mapping work が four-column source model と UAX \#60 / `SealSources.txt` へ整理される。

## 論点

### source caveats

N5211 は、source data をそのまま uniform な evidence として扱えないことを示す。DYC column では Kanji Database Project 側の digital data に 6 characters の欠落があり、空欄は source tradition の欠落ではなく digital data の問題だと説明する。K 系では Xiao Xu ben volume 25 が失われ、残存 copies は対応する X content で穴埋めしている点を caveat とする。

### font-derived columns

華東、北師、逢甲などの columns は font-derived data を含む。documentation は、その character set や source basis が完全には明示されない場合があること、逢甲 font は corresponding CJK Unified Ideographs によって code point が決まるため duplicate glyphs が 1 glyph にまとめられることを記録する。

### unification rules の素材

文書は、`二` / `丄` / `上` や inverted form の例を使い、各 source が同じ数の forms を持たなくても、複数 source の structurally different forms が unification rules の検討材料になると説明する。この観点は、後続の [WG2 N5307R](wg2-n5307r.md) の mapping / unification principles に接続する。
