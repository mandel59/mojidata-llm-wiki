# L2/26-104

## 要約

`L2/26-104` は、Proposed Update UTS \#51 Revision 30 "Unicode Emoji"、Version 18.0 draft 1 である。

UTS \#51 は emoji characters / sequences の構造、emoji data、text / emoji presentation、skin tone、ZWJ sequences、tag sequences、RGI emoji set、design guidance を定義する。

Revision 30 は Emoji 18.0 向けに versions / dates を更新し、implementation examples を変更する proposed update として登録された。

## 提案内容

文書は Unicode emoji の仕様本体であり、Emoji 18.0 の draft specification として、次のような領域を束ねる。

| 領域 | 内容 |
| --- | --- |
| definitions | emoji character、emoji presentation sequence、emoji tag sequence、emoji ZWJ sequence、RGI emoji set など。 |
| design guidance | names、gender-neutral emoji、skin tones、hair components、color、glyph facing direction、ZWJ sequence order。 |
| data files | `emoji-data.txt`、`emoji-sequences.txt`、`emoji-zwj-sequences.txt`、CLDR emoji data など。 |
| conformance | RGI / qualification、valid flag / tag sequences、unsupported or invalid sequences の fallback guidance。 |

Revision 30 の modifications は、version / dates の更新と Section 2.4.1 implementation example の変更が中心である。

## 後続決定

[L2/26-098](utc-l2-26-098.md) は ESR report として Emoji 18.0 feedback、interoperability、CLDR keyword intake を扱う。UTC \#187 agenda [L2/26-092](utc-l2-26-092.md) では UTS \#51 proposed update / PRI \#543 が emoji 議題に含まれる。

## 論点

### RGI と implementation guidance

UTS \#51 は単に repertoire を列挙するだけではなく、どの sequences を RGI として一般交換に推奨するか、invalid / ill-formed tag sequences をどう見せるか、ZWJ fallback をどう扱うかを定義する。Emoji interoperability topic では、文字追加よりもこの behavior data が重要になる。

### CLDR との同期

Emoji names、short names、keyword search、ordering は CLDR data と密接に接続する。`L2/26-104` は UTS \#51 側の proposed update であり、ESR report の CLDR keyword intake 議論と合わせて読む必要がある。
