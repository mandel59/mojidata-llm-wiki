# L2/07-159

## 要約

`L2/07-159` は、U-source characters を UTC 内部 database や過去の L2 dumps に閉じず、public / versioned document として公開することを提案した文書である。CJK Extension C の review で、U-source characters の存在や source attribution correction を WG2 participants が追跡しにくかったことを問題にしている。

文書は、John Jenkins と Richard Cook が保守していた FileMaker database に、index、glyph、source information、IDS、CDL data など IRG に必要な fields が含まれると説明する。indices は再利用されず、後続 versions は前 versions の superset になるという stable identifier policy も示されている。

## 提案内容

文書の提案は、U-source database 全体と relevant fields を含む UTR document を作り、Unicode Web site で公開することである。これにより、Unicode Standard や ISO/IEC 10646 から U source へ normative reference でき、関係者が character sources を追跡できるようにする。

この提案は、後続の [L2/08-284](utc-l2-08-284.md) / UTR #45 で具体化される。UAX #45 の `UTC-00001` 形式の identifiers、glyph / data files、status / source fields は、この文書で示された public reference 化の要求に対応している。

## 後続決定

この文書単独を取り上げた UTC meeting minutes はまだページ化していないが、提案の outcome は後続の [L2/08-284](utc-l2-08-284.md) / Draft UTR #45 に具体化されている。以後の UAX #45 は、U-source database を public / versioned reference として公開するというこの提案の方向を引き継いでいる。

## 論点

`L2/07-159` の重要点は、U-source を単なる候補一覧ではなく、WG2 / IRG review と ISO/IEC 10646 の source attribution が参照できる stable record として扱ったことである。後続の UAX #45 はこの public record としての性質を引き継ぎつつ、未符号化候補、encoded ideographs、variant / rejected records を同じ data model で管理する。
