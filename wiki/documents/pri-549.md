# PRI 549

## 要約

`PRI #549` は、UTC が Unicode 18.0.0 向けの UAX \#42 Unicode Character Database in XML Revision 39 proposed update を public review に出した PRI である。PRI page の status は Closed、closing date は 2026-07-07。

PRI page は、UCD の properties / values に合わせて new attributes が追加され、deprecated properties と historical information 用の attributes が削除されたと説明する。

UAX \#42 Revision 39 の変更は、Unicode 18.0 の new scripts / blocks、Jurchen / Seal / Tangut data、Unihan property changes を XML representation に反映するものである。Small Seal や UAX \#60 data を機械処理する場合、UAX \#44 / UAX \#60 だけでなく UAX \#42 schema も確認対象になる。

## 提案内容

### UAX \#42 Revision 39

Revision 39 は Unicode 18.0.0 の `age=18.0`、new blocks、new script values、new joining group valuesを XML attributes / values として反映する。

ideographic data に関係する主な追加は、Jurchen attributes、Seal attributes、Tangut numeric attribute、Unihan の `kJapaneseNewVariant` / `kJapaneseOldVariant` である。削除側では、Unihan の `kIRGDaeJaweon` / `kIRGKangXi` attributes が removed code point attributes として扱われる。

### Seal / UAX \#60 data

UAX \#42 draft は Seal data の XML attributes として `kSEAL_CCZSrc`、`kSEAL_DYCSrc`、`kSEAL_MCJK`、`kSEAL_QJZSrc`、`kSEAL_Rad`、`kSEAL_THXSrc` を記録する。これは [UAX \#60 Data for Large East Asian Scripts](../topics/uax60-large-east-asian-scripts.md) が扱う Seal source / radical data を XML UCD 表現へ写す部分である。

## 後続決定

`PRI #549` は 2026-07-07 に closed になっている。feedback の処理結果は UTC \#188 minutes / action items、最終 UAX \#42、最終 UCD XML schema / `ucdxml` data で確認する必要がある。

## 論点

### UCD text files と XML schema の同期

Unicode 18.0 の property 追加は、plain text data files、UAX \#44 documentation、UAX \#42 XML representation の 3 層で同期する必要がある。`PRI #549` は、実装者が XML form の UCD を取り込む場合の schema compatibility checkpoint である。

### Small Seal / large East Asian scripts

Small Seal の encoding proposal は repertoire と source evidence の議論だが、Unicode 18.0 の data publication では Seal source references / modern CJK reference / radical data が UAX \#60 と UAX \#42 に現れる。したがって Small Seal は proposal page だけでなく release data schema の変化としても追う必要がある。

### Unihan property lifecycle

`kJapaneseNewVariant` / `kJapaneseOldVariant` の追加と、`kIRGDaeJaweon` / `kIRGKangXi` の削除は、Unihan data format / property lifecycle の問題である。XML schema で removed attributes になるため、downstream tool は unknown attribute 追加だけでなく attribute removal にも備える必要がある。
