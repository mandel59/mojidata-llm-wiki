# L2/26-137

## 要約

`L2/26-137` は、left-to-right joining scripts に Joining_Type property をどう割り当てるかを検討する文書である。N'ti と Ndiko Jonam のような joining LTR scripts が提案されたことで、visually right-joining characters を `L` とするか `R` とするかが問題になる。

文書は、既存の Phags-pa U+A872 の扱いとの互換性を重視する案と、left / right を視覚方向どおりに再整理する案を比較する。著者は specification としては後者、つまり U+A872 を `R` に変える方向が clean と見る。

## 報告内容

| Option | 内容 | tradeoff |
| --- | --- | --- |
| Left is Right, Right is Left | 既存 U+A872 の practice を維持し、LTR script の visually right-joining を `L` とする。 | backward-compatible だが、property name と説明が混乱しやすい。 |
| Left is Left, Right is Right | U+A872 を `R` に変え、LTR script の visually right-joining を `R` とする。 | specification は明快だが、既存 Phags-pa implementations への影響がある。 |

## 論点

### property semantics と既存実装

Joining_Type は元々 Arabic shaping の文脈で発達した property であり、top-to-bottom scripts への解釈は既に例外を持つ。LTR joining scripts を追加する場合、property value の名前、Core Specification text、ArabicShaping.txt header、既存 shaping implementations の互換性を同時に考える必要がある。
