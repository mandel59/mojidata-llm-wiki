# WG2 N5301

## 要約

`WG2 N5301` は、2025-06-23/27 に Niigata, Japan で開催された WG2 Meeting \#72 の minutes and action items である。この wiki では、Section 8.2 の Japan IRG active participation request と、Section 11 / M72.07 の J-source glyph revert を主に参照する。

Section 8.2 は `WG2 N5284` を入力文書として、Japan が IRG に active member body として戻るべきかを議論した。minutes は、36,000 件超の horizontal extension `WG2 N5221` は WG2 より IRG に提出した方がよいという文脈を記録する。

Section 11 は `WG2 N5296` を入力文書として、JK-source / J-source chart glyphs が font change により HeiseiMincho / MS Mincho から IPAmjMincho へ変わって見える問題を扱い、Project Editor が J-source glyph changes を revert する disposition を記録する。

## 決定・Action Item

| 節 | 内容 | 関連 |
| --- | --- | --- |
| 8.2 | Japan の IRG active participation request を議論。Japan representative は復帰を検討できるが human resources / workload が課題だと述べた。 | [IRG N2721](irg-n2721.md), [WG2 N5221](wg2-n5221.md), [JMJ Horizontal Extension Review Path](../topics/jmj-horizontal-extension-review-path.md) |
| 11 | `WG2 N5296` の glyph changes を議論。Michel Suignard は first part of `WG2 N5296` に基づいて fix することに同意し、Japan が all J sources を含む single font を提供できると有用だと述べた。 | [WG2 N5296](wg2-n5296.md), [J-source and JMJ Source Issues](../topics/j-source.md) |
| M72.07 | WG2 は Project Editor が `WG2 N5296` に記載された J-source glyph changes を revert するよう勧告した。 | [WG2 M72.07 J-source glyph revert recommendation](../events/wg2-m72-07-j-source-glyph-revert.md), [WG2 N5304](wg2-n5304.md) |

## 後続決定

[WG2 N5304](wg2-n5304.md) は、M72.07 を formal recommendation として記録している。wiki では、J-source glyph revert の canonical event を [WG2 M72.07 J-source glyph revert recommendation](../events/wg2-m72-07-j-source-glyph-revert.md) に集約する。

Japan IRG active participation については、`WG2 N5301` 時点では復帰決定ではなく、human resources / workload が課題として残る。`IRG N2859` / `IRG N2870` は、その後も Japan NB の source owner response が必要になることを示す。

## 論点

### review path と correction issue の接続

Section 8.2 は、large JMJ horizontal extension の review body を IRG に寄せる必要を扱う。一方、Section 11 と M72.07 は、JMJ-source expansion 後の code chart font selection による glyph stability issue を扱う。両者は関連するが、前者は review workflow、後者は post-extension chart correction である。

### Japan NB の workload

minutes は、Japan が復帰を検討できる一方で human resources / workload が課題だと記録する。これは、large horizontal extension、J-source correction、IRG working set review を Japan NB が source owner としてどう処理するかという継続論点である。
