# L2/26-096

## 要約

`L2/26-096` は Properties & Algorithms Group が UTC Meeting \#187 に提出した report / recommendations である。Unicode 18.0 に向け、UCD properties、line breaking、segmentation、IDNA、collation、security、confusables、math、UTS \#58、UTS \#61 などを横断して処理する。

この文書は個別の annex / UTS proposed update だけでは見えにくい、data file と algorithm text の整合調整を束ねる。特に line break property、Grapheme Cluster Boundary、Indic_Conjunct_Break、UCA weights、UTS \#39 security data、confusables data が実装者向けの確認点になる。

## 決定・Action Item

| 区分 | 要点 |
| --- | --- |
| Variation selector conformance | `L2/26-070R` については revised document が誤った DICP definition を取り下げたため、UTC \#187 では action せず、後続改訂で扱う。 |
| UAX \#57 / East Asian data | `kEH_FVal` delimiter を `U+007C` に統一し、`/` との併記をやめる。 |
| UCD property updates | Arabic marks などに Diacritic property を付与し、U+FE51、U+2012、U+2013、U+00AD の Line_Break 値と LB12a text を更新する。 |
| Segmentation | UAX \#29 GB9c と UAX \#44 の InCB=Linker derivation を新規文字に合わせて調整する。 |
| IDNA | IDNA section の `CONTEXT0` typo を修正する。 |
| Collation | UTS \#10 / DUCET / CLDR の alignment として Tibetan contractions、U+FFFE / U+FFFF weights などを調整する。 |
| Security / confusables | UTS \#39 の casefolding wording、Identifier_Type、mid-priority confusables data、Cyrillic Extended-D などを処理する。 |
| UTR \#59 | East Asian Spacing は visible spacing、language context、UCD property derivation を含む Properties & Algorithms issue として扱われる。 |
| UTS \#61 | Unicode Set Notation を draft status に進める。 |

## 後続決定

UTC \#187 minutes [L2/26-093](utc-l2-26-093.md) が、PAG report を Unicode 18.0 beta review の action items として扱う。個々の public review issue は [L2/25-100](utc-l2-25-100.md)、`L2/26-109`、`L2/26-110`、[L2/26-111](utc-l2-26-111.md) などの proposed update 文書にも分散する。

## 論点

### algorithm text と data file の同期

PAG report の中心は、新規文字の追加そのものではなく、UCD data、annex text、implementation data が同じ結果を返すように調整することである。Line_Break、Grapheme Cluster Boundary、Indic_Conjunct_Break、UCA weights は、文字追加と同時に実装差が出やすい箇所として扱われている。

### security data の優先順位

UTS \#39 / confusables では、CJK confusables だけでなく Cyrillic Extended-D、compatibility equivalent sequences、mid-priority data も同じ report に入る。Unicode 18.0 の security review は、script 固有の topic だけでなく [Unicode Properties and Algorithms](../topics/unicode-properties-and-algorithms.md) から追う必要がある。
