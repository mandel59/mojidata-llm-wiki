# IRG N2893

## 要約

`IRG N2893` は、China による CJK Hybrid Characters の encoding policy 文書である。文書は、Han ideographs と他 script / symbol からなる characters、または Kana / Bopomofo / Latin / symbol だけから成るが CJK text で使われる characters を、"CJK Hybrid Ideographs" ではなく "CJK Hybrid Characters" と呼ぶべきだとする。

China は、新しい CJK Hybrid Characters block を設け、Han-Kana、Han-Bopomofo、Han-Latin などは CJKUI に入れない方針を主張する。一方、Han-Hangul hybrid characters については、font style と usage が Han ideographs と概ね一致するため、CJKUI に引き続き入れ得るとする。

## 決定・Action Item

| 区分 | 内容 |
| --- | --- |
| 名称 | "CJK Hybrid Ideographs" ではなく "CJK Hybrid Characters" を推奨。 |
| Block policy | CJK Hybrid Characters block の新設を推奨。 |
| CJKUI inclusion | Han-Hangul は CJKUI に継続可能。Han-Bopomofo / Han-Kana / Han-Latin などは CJKUI に入れるべきでない。 |
| Procedure | submitter はまず IRG に supporting information を提出し、IRG が CJKUI に入れるか CJK Hybrid Characters block に送るかを判断する。 |

## 論点

### Han-Kana / Han-Bopomofo の除外

文書は、Bopomofo、Hiragana、Katakana が Han ideographs に由来する面を認めつつ、独自の application scenarios と font styles を持つため、Han-Kana / Han-Bopomofo hybrid characters は CJKUI に入れるべきでないとする。特に、゜（U+309C）や ㆱ（U+31B1）のように歴史的な Han ideographs に現れない部品を含むことを理由に挙げる。

### IRG の gatekeeping

China は、CJK Hybrid Characters の提出をまず IRG で議論し、IRG が CJKUI に含めると unanimous に判断したものだけを次の IRG working set に送るべきとする。それ以外は WG2 へ CJK Hybrid Characters block として提出する。
