# WG2 N5307R

## 要約

`WG2 N5307R` は、TCA and China による "Mapping Principles of Small Seal Glyphs in Different Shuowen Versions" の revised document である。

文書は、Tenghuaxie version (THX)、Chenchangzhi version (CCZ)、Xiaoxu Ben / Qi Junzao version (QJZ)、Duan Zhu version (DYC) の Small Seal glyphs を、どの条件で同一 code point に unified し、どの条件で disunified するかを整理する。

この文書は [WG2 N5318R](wg2-n5318r.md) の four-column proposal と [WG2 N5341](wg2-n5341.md) の code charts / data set、後続の [WG2 N5344R2](wg2-n5344r2.md) の revised proposal を読むための rules document である。

## 報告内容

### General principles

THX と CCZ はどちらも Daxu Ben 系で、glyph、explanation、ordering が概ね一致する。一方、QJZ と DYC は一部の character interpretation が Daxu Ben と異なり、form、explanation、ordering の差を生む。

Coding sequence は、THX の Zhengzhuan（正篆）、Chongwen（重文）、newly added characters（新附字）の順序を基準にする。THX に対応しない forms は THX の最後に続け、CCZ、QJZ、DYC の順で補う。

### Written form の境界

Small Seal script の written forms には Zhengzhuan（正篆）、Guwen（古文）、Zhouwen（籀文）があり、unification principle は同じ style system 内の関係だけを扱う。異なる style systems に属する characters は、glyph が似ていても別 characters として処理する。

### Unification principles

文書は次のような場合に unified とする。

| Principle | 内容 |
| --- | --- |
| 2.(1) | taboo glyphs など、避諱や版本差による glyph variation。 |
| 2.(2a) | glyph が explanation と矛盾しており、incorrect seal form と判断できる場合。 |
| 2.(2b) | component shape が同一 source 内の system consistency から外れる場合。 |
| 2.(3) | `門` や `言` など、source ごとに components が systemically modified される場合。 |
| 2.(4) | compound characters の minor differences / variant forms。 |
| 2.(5) | phono-semantic compound characters で semantic / phonetic components の配置差があっても meaning が同じ場合。 |
| 2.(6) | Guwen / Zhouwen で correctness を確認しにくく、説明が不足する場合は新規 code point を作らず unified に保つ。 |

### Disunification principles

文書は次のような場合に disunified とする。

| Principle | 内容 |
| --- | --- |
| 3.(1) | pronunciation / meaning が同じでも components が異なる場合。 |
| 3.(2) | 同じ components と同じ component count を持つ compound characters でも structure が異なる場合。 |
| 3.(3) | single characters で pronunciation / meaning が同じでも stroke count や stroke relationship が異なる場合。 |

## 後続決定

[WG2 N5304](wg2-n5304.md) の M72.13 は、`WG2 N5307R` を直接の recommendation 文書としては列挙しないが、[WG2 N5318R](wg2-n5318r.md) と [WG2 N5341](wg2-n5341.md) は N5307R を mapping / unification principles の根拠として位置づける。後続の [WG2 N5344R2](wg2-n5344r2.md) は、この principles section を proposal text に取り込み、actual code chart / database に反映した。

## 論点

### 同じ形か、同じ文字か

`WG2 N5307R` の中心は、glyph similarity だけではなく、source tradition、written form、explanation、component system consistency を合わせて character identity を決める点である。Small Seal proposal では、現代 CJK で同じ字に見えるかよりも、`Shuowen Jiezi` の版本間で同じ抽象文字として扱えるかが重要になる。

### N5344R2 での実装

N5344R2 の four-column chart と database は、N5307R の principles を実際の repertoire / source references / radical grouping に落としたものとして読む。後続の CD.4 disposition で `kSEAL_MCJK` や `kSEAL_Rad` が修正された場合も、principles と actual data のどちらに由来する修正かを分けて見る必要がある。
