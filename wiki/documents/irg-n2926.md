# IRG N2926

## 要約

`IRG N2926` は、Singapore characters を既存の `kIRG_GSource` 内の `GS` prefix から分離し、新しい normative property `kIRG_SGSource` を設ける提案である。Singapore が IRG encoding work に参加していない状況で、GS-source glyph issues を誰が処理するかが曖昧になることを問題にする。

文書は、`SG-[0-9A-F]{4}` syntax を持つ `kIRG_SGSource` の新設、`kIRG_GSource` syntax / description から `GS` prefix を除くこと、既存 GS-prefixed references を SG-prefixed references へ移すことを求める。U+48B4 / GS-2151 の glyph issue を例に、Singapore source を G-source から分ける必要を説明する。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| new property | `kIRG_SGSource`。status は Normative、category は IRG Sources、syntax は `SG-[0-9A-F]{4}`。 |
| G-source change | `kIRG_GSource` の syntax / description から `GS` source prefix を削除。 |
| migration | 現在の GS-prefixed G-source characters を SG-source references へ移す。 |
| U+48B4 | original GS-2151 glyph を SG-2151 glyph として U+9097 へ移し、China が同意すれば U+48B4 の G-source reference を `GKX-1268.18` に変更する案。 |

## 後続決定

[IRG N2909](irg-n2909.md) の M66.16 は `kIRG_SGSource` establishment と GS prefix の SG source への移行を勧告した。[WG2 N5354](wg2-n5354.md) の M73.15 は、この proposal を `WG2 N5359` として扱い、Amendment 1 target に追加するよう勧告した。[IRG N2935](irg-n2935.md) は、WG2 と SC2 が受け入れ、Unicode Version 19.0 target としている current status を記録する。

## 論点

### source prefix は governance を表す

`IRG N2926` は、source prefix が単なる文字列ではなく、glyph issue や source reference changes の責任主体を表すことを示す。Singapore が IRG に参加していない状態で `GS` を G-source に置き続けると、China が処理できない Singapore-specific issue が G-source governance に残る。
