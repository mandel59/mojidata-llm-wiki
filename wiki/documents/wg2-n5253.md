# WG2 N5253

## 要約

`WG2 N5253` は、ISO/IEC 10646 7th edition を、Unicode Standard / UAX / UTS / UCD data への normative reference を増やすことで安定化する検討文書である。Unicode と ISO/IEC 10646 の repertoire が近年ほぼ同期しているため、可変 data を二重に維持せず、Unicode 側の data / specification を参照する方向を示す。

直接の前例として ISO/IEC 14651 7th edition CD が挙げられる。ISO/IEC 14651 は UTS \#10 Unicode Collation Algorithm と同期するため、Common Template Table を UTS \#10 側で提供し、標準本文を安定化できるという model である。

## 報告内容

| 区分 | 内容 |
| --- | --- |
| goal | ISO/IEC 10646 本文の stable content と variable repertoire-dependent content を分離し、可変 data は Unicode 側の specification / data set への normative reference に寄せる。 |
| property data | Combining Classes、General Category、Bidi Mirrored、Age などは既に Unicode data files への normative reference を持つ。 |
| attachment data | UCSVariants.txt、EmojiSrc.txt、CJKSrc.txt、NushuSrc.txt、TangutSrc.txt、NUSI.txt、Allnames.txt などには Unicode 側の direct equivalent がある。 |
| clauses | CJK source references は UAX \#38、pictographic symbols は UAX \#51、Egyptian Hieroglyph source references は UAX \#57 のように、より詳細な Unicode 文書を参照できる可能性がある。 |

## 後続決定

[WG2 N5315](wg2-n5315.md) はこの検討を code charts の共通資源化へ進め、WG2 \#72 M72.10 で Project Editor が ITTF と external / linked chart publication model を検討する勧告につながった。後続の [WG2 N5363R](wg2-n5363r.md) と WG2 \#73 M73.03 は、edition-specific code charts を external normative reference とする具体案へ進めている。

## 論点

### Stable text と variable data の分離

`WG2 N5253` の主眼は、ISO/IEC 10646 の全内容を毎回本文内で複製するのではなく、Unicode と同期する部分を external data / annex へ寄せることで、標準本文を安定させる点にある。これは code charts の外部化だけでなく、UCD property data、source reference data、names data の重複保守を減らす提案である。

### Unicode 側文書の整備要求

Tangut や Nüshu の source data は UAX \#44 に短く説明されるだけで、UAX \#38 の Unihan や UAX \#57 の Unikemet ほど formal な説明がないとされる。ISO/IEC 10646 から normative reference するには、Unicode 側の document / data format 説明を ISO reference として使いやすい形に整える必要がある。
