# PRI 533

## 要約

`PRI #533` は、UTC が UTR \#25 Unicode Support for Mathematics Revision 16 draft を public review に出した PRI である。PRI page は status を Open、closing date を 2026-07-07 と表示している。2026-07-10 時点では、catalog も `review_status: open` としているため、この wiki では未処理の open review として扱う。

review 対象は [L2/25-267](utc-l2-25-267.md) / UTR \#25 Revision 16 draft 2 で、base Unicode version は Unicode 17.0.0、date は 2025-10-02。PRI page は、今回の revision が主に PDF から HTML への conversion、accessibility、navigation、他の Unicode Technical Reports との consistency を改善する構造変更であると説明する。

ただし PRI page は、今後の significant content revisions として、Unicode 9 support から Unicode 17 support への更新、新規 characters / properties の取り込み、examples / tables の改訂、current best practices への alignment を予定していることも示す。

## 提案内容

### UTR \#25 Revision 16 draft

UTR \#25 は、Unicode の mathematical character repertoire、mathematical alphanumeric symbols、math character properties、input / programming language / expression recognition guidance、math data files、security considerations を扱う informative technical report である。

Revision 16 の modifications section は、主な変更として次を記録する。

- report を HTML に変換した。
- math data files を W3C HTML-MathML Entity definitions に合わせ、Unicode General Category を listed data に追加した。
- Section 3.1.3 Multiple Use を追加した。
- Section 3.2.7 Combining Marks に single text runs の説明を追加した。
- internal / external links、references、character listing consistency を整理した。

### data files

UTR \#25 draft は、数学文字を primary usage で分類する `MathClass-16.txt` と、SGML / MathML などの entity sets への mapping を持つ `MathClassEx-16.txt` を data files として参照する。draft は、`Math` property を持つ全 characters が classification に含まれ、formal default `Math_Class` assignments はないと説明する。

## 後続決定

`PRI #533` は closing date を過ぎているが、2026-07-10 時点で PRI page / catalog は Open のままである。UTC \#188 の feedback handling、最終 UTR \#25、math data files の revision-16 publication を確認する必要がある。

この PRI は個別の Leibnizian / historic mathematical symbols proposal の採否ではない。UTR \#25 は現代的な mathematical text support の overview / data file / implementation guidance であり、[Leibnizian and Historic Mathematical Symbols](../topics/leibnizian-and-historic-mathematical-symbols.md) の個別 proposal 群とは別の layer として読む。

## 論点

### HTML migration と content update の段階差

今回の draft は HTML migration が中心で、Unicode 17 までの内容更新は今後の significant content revisions として予告されている。したがって、Revision 16 draft を「Unicode 17 まで完全更新済み」と読むのは早い。HTML 化により、後続で data / examples / references を保守しやすくする段階と見る。

### mathematical text の plain text boundary

UTR \#25 は MathML / OpenMath / TeX / UnicodeMath と Unicode plain text の関係を説明し、mathematical alphanumeric symbols が単なる font style ではなく数式内の semantic distinction を保持する場合を扱う。この点は、historic mathematical symbols や COMPOSE / overstriking のような plain text boundary 論点と接続する。

### security

UTR \#25 draft は、数学文字が regular text characters と confusable になり得るため、mathematical alphanumeric symbols が domain names など security sensitive environments では除外されると説明する。これは [Unicode Properties and Algorithms](../topics/unicode-properties-and-algorithms.md) の UTS \#39 / identifier policy と隣接する。
