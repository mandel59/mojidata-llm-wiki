# IRG N2953

## 要約

`IRG N2953` は、[IRG N2928](irg-n2928.md) で提起された U+268A1 / U+268A2 の V-source representative glyph issue への follow-up である。IRG \#66 で Vietnam NB は現行の未符号化 component が意図されたものだと回答したが、この文書は glyph change を再検討するための追加 evidence を示す。

文書は、現行形を支持する証拠として `Bảng tra chữ Nôm`（1976）を挙げる一方、`Giúp đọc Nôm và Hán Việt`（2004）には U+268A1 に対応する normalized form ⿰屋脊 が見つかるとする。また、U+26860 は `Bảng tra chữ Nôm` では現行 issue と同系の component を持つが、Unicode では ⿰弄脊 に normalized されていると述べる。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| 前提 | `IRG N2928` に対し、Vietnam NB は現行 component が意図されたものだと回答した。 |
| 現行形の証拠 | U+268A1 / U+268A2 の現行 component は `Bảng tra chữ Nôm` に見えるため、Extension B encoding 時の evidence だった可能性がある。 |
| 追加証拠 | `Giúp đọc Nôm và Hán Việt` に、U+268A1 に対応する ⿰屋脊 の normalized form がある。 |
| 比較証拠 | U+26860 は `Bảng tra chữ Nôm` で同系の component を持つが、Unicode では 脊 component に normalized されている。 |
| 結論 | 現行 component と 脊 は unifiable variants として相互に使われている可能性があり、Vietnam NB が U+268A1 / U+268A2、少なくとも U+268A1 の glyph change を検討する根拠になる。 |

## 後続決定

[IRG N2935](irg-n2935.md) は、Meeting \#67 agenda Section 9.1 で `IRG N2928` を "Redux" issue とし、その follow-up として `IRG N2953` を挙げている。会合前 agenda なので、実際の採否や action item は Meeting \#67 recommendations `IRG N2946` が available になった時点で確認する必要がある。

## 論点

### 相反する evidence の扱い

`IRG N2953` の重要点は、現行 glyph を支持する evidence と、脊 component への normalized form を支持する evidence の両方を認めていることである。これは単純な誤字修正ではなく、source evidence、normalization practice、Vietnam NB の意図確認を合わせて判断する representative glyph issue である。

### Existing code point の glyph update

この文書は U+268A1 / U+268A2 を再符号化する提案ではなく、既存 code point の V-source representative glyph を変えるかどうかを問う。したがって [IRG Source Data and Representative Glyphs](../topics/irg-source-data-and-representative-glyphs.md) では、source reference update ではなく glyph evidence review の事例として扱う。
