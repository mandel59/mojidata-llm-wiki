# L2/24-165

## 要約

`L2/24-165` は、2024-07-05 の CJK & Unihan Working Group meeting に基づく UTC Meeting \#180 向け recommendations である。UTC \#179 以降の PRI \#483 feedback、public feedback、WG2 \#71 recommendations、UAX \#45 proposals、その他 CJK-adjacent proposals を扱う。

文書は、`kMandarin` additions / changes を CLDR-TC に再確認させる action items、`kDefinition` / `kGB3` fixes、CJK Unified Ideographs Extension J と U+5CC0 disunification の Unicode 17.0 target、UAX \#45 `FutureWS` additions、UTC submission for IRG Working Set 2024 の FYI をまとめた。

## 決定・Action Item

| 区分 | 決定・Action Item |
| --- | --- |
| `kMandarin` review | U+3235B / U+3236D、U+2E146、U+4680、U+941F などの proposed `kMandarin` additions / changes を CLDR-TC に確認させる action items を置いた。 |
| Unihan property fixes | U+9527 の `kDefinition` を corrected value に変更し、U+53D1 / U+5FF9 / U+72AE / U+225D6 の `kGB3` values を remove / add するよう勧告した。 |
| Extension J / U+5CC0 | WG2 \#71 recommendations に基づき、4,300 characters の CJK Unified Ideographs Extension J と U+5CC0 disunification / U+2B73A encoding を Unicode 17.0 target として受け入れるよう勧告した。 |
| UAX \#45 / U-source | Korean place name 1 record UTC-03488 と Daoist talisman 23 records UTC-03489..UTC-03511 を `FutureWS` として受け入れ、evidence images / descriptions を private repository で管理する運用を示した。 |
| IRG / WS2024 | UTC submission for IRG Working Set 2024 は 244 UAX \#45 ideographs と 794 evidence images からなる FYI document とされた。IRG \#63 以降の UNC proposals と新しい IRG document register の運用も報告された。 |

## 後続決定

[L2/26-084](utc-l2-26-084.md) は、[L2/24-012](utc-l2-24-012.md)、[L2/24-067](utc-l2-24-067.md)、`L2/24-165` の 2024 年 action items に由来する 169 characters の `kMandarin` additions / changes へ、CLDR TC と Apple Localization and Release Engineering が返した feedback である。[L2/26-099](utc-l2-26-099.md) Section 22 は、その feedback を Unicode 18.0 target の update として進めるよう勧告した。

## 論点

### Unicode 16.0 / 17.0 / 18.0 の時点差

`L2/24-165` は Unicode Version 16.0 向けの Unihan property fixes、Unicode Version 17.0 target の Extension J / U+5CC0 disunification、IRG Working Set 2024 への UAX \#45 handoff を同じ recommendation bundle に含む。topic page では、どの update がどの Unicode version / IRG stage に属するかを分けて読む必要がある。

### CLDR-TC review の再確認

U+3236D や U+941F のように、過去の `kMandarin` value や CLDR-TC result と新しい evidence が食い違う場合、CJK & Unihan Working Group は直接上書きせず、CLDR-TC に再確認を求めている。`L2/26-084` はその review loop の後続文書である。

### UAX \#45 evidence management

UAX \#45 additions が WS2024 submission 規模に達したため、文書は新規 U-source ideographs の evidence images と descriptions を repository で管理する運用を示した。これは UAX \#45 を単なる registry ではなく、IRG submission preparation layer として扱う実務上の変化である。
