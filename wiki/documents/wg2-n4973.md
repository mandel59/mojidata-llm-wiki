# WG2 N4973

## 要約

`WG2 N4973` は、[WG2 N4853](wg2-n4853.md) が残した duplicate glyph と Tenghuaxie version（藤花榭本）の source compatibility に関する論点へ、TCA と China の専門家が具体的な分類結果を示した feedback である。

文書は Tenghuaxie version の重文について、15組を exact duplicate として各組1 code point に統合する一方、6組の possible duplicate は字形体系を保存するため別々に符号化する案を示す。また8件の taboo character deformation は、他の `Shuowen` versions を参照して本来の glyph を復元するとした。

これは WG2 / UTC に検討を求める proposal-status の working document であり、WG2 plenary の決定ではない。後続の [WG2 N5089](wg2-n5089.md) は、2019年の informal meeting で duplicate、glyph restoration、variation mechanism を再検討した結果を報告する。

## フィードバック内容

### Duplicate の分類

文書は `WG2 N4853` の三分類、すなわち exact duplicate、possible duplicate、non-duplicate を前提に、Tenghuaxie version の具体例を選別した。

- exact duplicate は15組で、線の態勢や長さの差が glyph structure に影響しないとして、各組につき1 code point のみを符号化する。
- possible duplicate は6組で、線数、線種、component position、Guwen / Zhouwen style などに小差がある。`Shuowen` を小篆字形体系の整理として捉え、各 glyph を個別に符号化する。
- non-duplicate は異なる character として別符号化するという、N4853 の整理を維持する。

この結論は、N4853 にあった exact duplicate と possible duplicate の双方を未確定の VS scheme で扱うという方向を、そのまま採用したものではない。N4973 は possible duplicate の6組には独立 code point が必要だと主張している。

### Taboo character deformation

Tenghuaxie version に8件の 避諱（taboo character deformation）があるとし、character substitution と stroke omission の2種類を挙げる。提案 data では Tenghuaxie glyph を無条件に保持せず、他の `Shuowen` editions を参照して変形前の glyph を復元し、符号化対象にする方針を示した。

文書末尾では、この分類と外部 feedback を用いて Shuowen Small Seal list を修正するとしており、repertoire、property、code chart の最終仕様を確定してはいない。

## 後続決定

この文書自体は disposition、recommendation、採択記録ではない。直後の `WG2 N4992` が N4973 にコメントし、`WG2 N4996` がそれへ応答したため、duplicate と glyph restoration は引き続き検討対象だった。

[WG2 N5089](wg2-n5089.md) は 2019-06-01 Beijing informal meeting の報告として、15組の Chongwen は各組1字のみを符号化する方針を維持し、非常に近い2組の representative glyph、誤刻・欠損、11件へ増えた taboo deformation の処理を整理した。ただし N5089 も informal consensus の報告であり、WG2 plenary による repertoire 採択ではない。

## 論点

### Source fidelity と encoded glyph

N4973 は Tenghuaxie version を repertoire の baseline としながら、taboo deformation については他版本による復元を求める。したがって baseline source の transcription と、standard code chart に載せる representative glyph を同一視しない立場である。

### Unification と字形体系の保存

exact duplicate では構造に影響しない差を統合し、possible duplicate では小差にも字形体系上の価値を認める。この境界が後続文書で検証される主要論点となった。
