# IRG N2960

## 要約

`IRG N2960` は、UK が提出した UK-source horizontal extension proposal である。IRG Working Set 2024 の review process で既存 CJK Unified Ideographs に unified された UK-source characters 2 件について、source reference と representative glyph を既存 code point へ追加することを求める。

対象は CJK Unified Ideographs Extension F の U+2D0E7 と U+2D3EB で、source references はそれぞれ `UK-30032` と `UK-30590` である。文書は、discussion records を根拠にした horizontal extension であり、font は PDF attachment に含まれると記録する。

## 提案内容

| Code point | Block | UK source reference | 論点 |
| --- | --- | --- | --- |
| U+2D0E7 | Extension F | `UK-30032` | WS2024 review で既存 ideograph と unified された UK-source character の source data 追加。 |
| U+2D3EB | Extension F | `UK-30590` | UK glyph は RS=11 / TS=13、UCS glyph は RS=12 / TS=14 で、stroke count 差を code chart と Unihan database のどちらに反映するかが論点。 |

## 後続決定

[IRG N2935](irg-n2935.md) は、IRG Meeting \#67 agenda Section 8 に [IRG N2960](irg-n2960.md) を UK horizontal extension proposal として載せている。2026-07-08 時点では Meeting \#67 は未開催であり、IRG の decision は未確認である。

## 論点

### source reference と glyph / stroke count の分担

Horizontal extension は新しい code point を追加しないため、source reference と representative glyph をどこまで code chart / Unihan data に反映するかが中心になる。U+2D3EB では UK glyph と UCS glyph の radical / total stroke count が異なるため、source-specific glyph difference を chart glyph と Unihan property のどちらで扱うかを IRG review で確認する必要がある。

### WS2024 review から horizontal extension へ

`IRG N2960` は、WS2024 review の結果として既存 ideograph に unified された candidate を、UK-source data として既存 code point へ接続する提案である。この点で [L2/26-147](utc-l2-26-147.md) / `IRG N2961` の U-source horizontal extension と同じ型の作業として読める。
