# IRG N2895

## 要約

`IRG N2895` は、Ma Shijie による `IRG N2878R` への feedback である。文書冒頭で、意見は複数人から来ており相互に矛盾し得るが、改変せずそのまま提示すると断っている。

主な論点は、incomplete excavated documents という使用場面、first batch の各 component に 3 件以上の CJK / Extension use case を求める品質基準、U+2FA20-U+2FC07 範囲の個別 component への delete / disunify / radical change コメント、GF 3001 / GF 0014 / GB/T 25741 / National Library of China / Zhonghua Book Company data からの漏れである。

## フィードバック内容

| 区分 | 内容 |
| --- | --- |
| usage scenario | incomplete excavated documents を新しい使用場面として挙げ、components が古籍・出土文献の欠損字や異体構造の記述に使われることを示す。 |
| quality criterion | first batch の各 component には CJK / Extension に 3 use cases が必要だとする。 |
| individual comments | U+2FA20-U+2FC07 に対して、source 間の disunification、delete、radical change、source origin の確認などを多数列挙する。 |
| lost components | GF 3001、GF 0014、GB/T 25741、National Library of China / Zhonghua Book Company data から proposal に漏れている component 候補を列挙する。 |
| encoding model question | Radical / Stroke / Component / CJK への重複的な符号化例を踏まえ、component として符号化後に CJK 用 evidence が出た場合に CJK でも再度符号化するのかを問う。 |

## 後続決定

[IRG N2878R3](irg-n2878r3.md) は individual contributions を review したうえで、first batch には採用しない component suggestions があると整理した。`IRG N2895` の多数の lost components / disunification comments は、この first batch と future batch の境界を考える feedback として読む必要がある。

IRG \#66 Recommendation M66.17 は [IRG N2909](irg-n2909.md) に記録され、`IRG N2878R3` の Components-A/B を WG2 へ forward した。同時に authors へ future CJK Unified Ideographs Components blocks へ components を追加提案できる方法の検討を求めたため、`IRG N2895` は後続 batch procedure の要求根拠にもなる。

## 論点

### source coverage と first batch policy

`IRG N2895` は、既に national standards が cited されているにもかかわらず GF 3001 / GF 0014 から漏れがあると指摘し、さらに GB/T 25741 が完全に omitted だとする。これは `IRG N2878R3` が first batch を固定する際に、complete source coverage よりも reviewable scope を優先したことと tension を持つ。

### component と CJK ideograph の境界

文書末尾の question は、radical、stroke、component、CJK ideograph を Unicode が別 block で重複的に符号化してきた事実を踏まえ、component として符号化した形が後に CJK 用 evidence を持つ場合の扱いを問う。これは [CJKV Components](../topics/cjkv-components.md) の core issue である「components as CJK ideographs」と、future batch / ordinary CJKUI encoding の境界に関わる。
