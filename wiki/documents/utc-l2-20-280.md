# A Proposal to Encode Taiwanese Kana

## 要約

`L2/20-280` は、Taiwanese Kana を UCS / Unicode に追加する proposal である。
提案は、Supplementary Multilingual Plane に Kana Extended-B block を設け、Minnan tone marks と nasalized tone marks を U+1AFF0..U+1AFFE に配置する案を示す。
Taiwanese Kana は、Taiwanese Hokkien / Min Nan を kana と tone marks で注記する historical writing system として説明される。
Tone 6 用の位置は将来証拠のために空け、U+0323 COMBINING DOT BELOW と U+0305 COMBINING OVERLINE は既存文字に統合する方針を示す。
この proposal は [L2/20-233](utc-l2-20-233.md) の feedback を取り込み、character names を Min Nan に寄せる方向へ改訂された前段文書である。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| block | Kana Extended-B、range U+1AFF0..U+1AFFF。 |
| tone marks | `KATAKANA LETTER MINNAN TONE-2` など、tone 2 / 3 / 4 / 5 / 7 / 8 を提案。 |
| nasalized tone marks | `KATAKANA LETTER MINNAN NASALIZED TONE-*` として tone 1 / 2 / 3 / 4 / 5 / 7 / 8 を提案。 |
| reserved positions | Tone 6 は現代 Hokkien で tone 2 と一致するため、U+1AFF4 / U+1AFFC を空ける。 |
| combining marks | aspiration mark は U+0323、overline は U+0305 に統合し、Taiwanese Kana 用の扱いを注記する。 |

## 後続決定

[L2/20-209R](utc-l2-20-209r.md) は revised final proposal として、Script Ad Hoc round two の変更まで受け入れた版である。現在の Unicode code chart では、Kana Extended-B は U+1AFF0..U+1AFFF に置かれ、Minnan Chinese を annotate する furigana extensions として説明される。

## 論点

### historical system と modern use

文書は、Taiwanese Kana を日本統治期に Japanese linguists が Taiwanese Hokkien を記録・教育するために整備した system として説明する。主要資料として Ogawa Naoyoshi の dictionaries、Mojikyo の legacy support、現代の digitized dictionary / Wikipedia usage が挙げられる。

### block naming と political / linguistic naming

Proposal は character names や block name に Taiwan を入れず、Minnan / Kana Extended-B とする方向を説明する。これは、Taiwanese Hokkien だけでなく Quanzhou dialect などにも使えること、地名・政治的呼称を character name に固定しないことに基づく。
