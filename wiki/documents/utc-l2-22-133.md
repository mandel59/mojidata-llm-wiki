# L2/22-133

## 要約

`L2/22-133` は、同一またはほぼ同一の glyph を持つが `kSpoofingVariant` または `kZVariant` property がまだ与えられていない CJK characters を列挙し、UTC に扱いを検討するよう求めた文書である。author は Ryusei Yamaguchi、本文日付は 2022-07-05、UTC registry 上の日付は 2022-07-06 である。

候補には `U+4E8C` / `U+2011E` / `U+2011F` / `U+20120`、`U+5341` / `U+2C09B`、`U+53F1` / `U+20B9F` / `U+2B738` などが含まれる。提案された property value changes and additions の data file が文書に添付されている。

## 論点

### kSpoofingVariant と kZVariant の境界

文書は spoofing concern と Z-variant relation の両方を扱う。例えば `緖` / `緒`、`謠` / `謡` は Z-variant pair だが `kZVariant` がないと注記し、`叱` / `𠮟` / `𫜸` では既存の `kSpoofingVariant` coverage に不足があると指摘する。

### 2026 年 review request への前段

`L2/26-127` は、`kSpoofingVariant` と `kZVariant` を CJK confusable pair 候補の生成に使っている。したがって `L2/22-133` は、2026 年の UTS \#39 `confusables.txt` review request に直接先行する Unihan property maintenance の文脈を与える。
