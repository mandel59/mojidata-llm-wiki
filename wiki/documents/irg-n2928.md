# IRG N2928

## 要約

`IRG N2928` は、V-source の U+268A1 と U+268A2 について、現行 representative glyph の component が意図されたものか、それとも U+810A 脊 に戻すべきかを照会する文書である。Ng Koon Hang は、両字が Vietnamese で "nóc" と読まれ、`nhà` と組んで「家屋の屋根の上」を意味することから、構成を 屋+脊 と見る可能性を示す。

文書は、Unicode 5.2 では両字が 脊 component を使っていた一方、Unicode 6.0 で現行形へ戻った理由を確認したいとしている。現行 component が印刷辞書で裏付けられるなら Vietnam NB が将来 working set へ component を提出する選択肢を示し、そうでなければ Unicode 5.2 と同じく 脊 component の glyph へ戻すことを提案する。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| 対象 | U+268A1 と U+268A2 の V-source representative glyph。 |
| 問い | 現行 glyph の component が意図されたものか、U+810A 脊 に戻すべきか。 |
| 根拠 | 両字は "nóc" と読み、「屋根の上」を表す語に使われるため、semantic component を 脊 と見る余地がある。 |
| 比較 | U+26860 と WS2024-04282 は、同じく "sống" / spine 系の意味で 脊 component を使う。 |
| 提案 | 現行 component が意図されたもので証拠があるなら future working set へ出す。そうでなければ U+268A1 / U+268A2 の glyph を 脊 component へ戻す。 |

## 後続決定

[IRG N2909](irg-n2909.md) の M66.15 は、Vietnam が original evidence を確認し、現行 representative glyph は意図されたものだとした一方、`IRG N2928` の glyph change suggestion は意味的には合理的であり、追加 evidence が必要だと整理した。十分な evidence が見つかれば Vietnam が glyph change を提案する、という扱いである。

[IRG N2953](irg-n2953.md) はこの follow-up として、現行形を支持する `Bảng tra chữ Nôm` と、脊 component への normalized form を示す `Giúp đọc Nôm và Hán Việt` の evidence を追加した。Meeting \#67 agenda [IRG N2935](irg-n2935.md) は Section 9.1 の "Redux" issue として `IRG N2928` / `IRG N2953` を継続議題に置いている。

## 論点

### Representative glyph と component encoding

`IRG N2928` は、新しい encoded ideograph を直接提案する文書ではなく、既存 V-source glyph の component をどう解釈するかを問う文書である。現行 component が独立した evidence を持つなら future working set の component / ideograph proposal につながり得るが、単に 脊 の variant と見なせるなら representative glyph update として処理できる。

### Evidence の重み

文書は Zi.tools を非公式な参考として扱い、判断の中心を Vietnam NB と IRG の evidence review に置いている。このため、topic 側では `IRG N2928` を決定ではなく、V-source representative glyph issue の発端として読む。
