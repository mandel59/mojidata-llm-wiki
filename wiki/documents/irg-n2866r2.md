# IRG N2866R2

## 要約

`IRG N2866R2` は、script-hybrid characters を通常の CJK Unified Ideographs ではなく、独立した CJK Hybrid Characters block に置くことを提案する文書である。R2 は IRG #65 での China、ROK、個人 expert の feedback を反映し、IRG review と UTC coordination の下で別 block を設ける案を再提示している。

文書は、Han-Katakana、Han-Hiragana、Han-Bopomofo、Han-Latin、all-Katakana、all-hiragana などを例示し、これらを ordinary ideographs として扱うには radicals、stroke counts、variants、unification rules の研究と consensus が不足しているとする。

## 決定・Action Item

| 区分 | 内容 |
| --- | --- |
| 提案 | CJK Hybrid Characters を独立 block に符号化する。 |
| Governance | IRG review と UTC coordination のもとで扱い、IRG P&P の大幅変更は不要とする。 |
| GB 18030 | IRG consensus が得られない場合、China が GB 18030 で unmapped approach を採ることも選択肢として示す。 |

## 論点

### 例示された hybrid cases

文書は、Han-Hangul U+439B、withdrawn / unencoded の Han-Katakana `⿸广マ`、CJK Extension F の Han-Hiragana U+2D92A、CJK Extension B の Han-Bopomofo U+20B9A、withdrawn / unencoded の Han-Latin `⿸广K`、CJK Extension E の all-Katakana U+2BCCD、CJK Extension F の all-hiragana U+2CF00 などを挙げる。

また U+323BF について、Latin X に由来する intent がありつつ code chart では Han component 㐅 として描かれた case と説明し、already encoded legacy cases の扱いが未解決であることを示す。

### CJKUI に入れない理由

CJKUI に入れると、CJK Unified Ideographs blocks が Han strokes / radicals だけからなるという実装者の前提が崩れる。文書は、非 Han letters を components として扱う方法、alphabetic shapes の stroke count、IDS syntax の拡張などが未整備のまま ordinary ideographs として符号化するのは危険だとする。
