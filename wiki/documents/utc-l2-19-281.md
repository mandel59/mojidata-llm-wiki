# L2/19-281

## 要約

`L2/19-281` は、Unihan database の `kZVariant` field に入れる data として 93 pairs of ideographs を提案した文書である。author は John H. Jenkins、文書本文の日付は 2019-07-18 である。

文書は、listed pairs を `kZVariants` として confidently labeled できる preliminary data set と説明し、later submission で増補され得るとしている。提案は、appropriate symmetrical relationships を持つ形で Unihan database に追加し、既存の `kZVariant` field content を置き換えることを求める。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| 提案 | 93 ideograph pairs を `kZVariant` data として Unihan database に追加する。 |
| 関係 | pair は symmetrical relationships として扱う。 |
| Scope | 既存の `kZVariant` field content を置き換える preliminary data set。 |
| 対になる文書 | 同日提出の [L2/19-282](utc-l2-19-282.md) は `kSpoofingVariant` field の初期 data proposal。 |

## 論点

### Z-variant data の置き換え

`kZVariant` は CJK ideographs の Z-variant relationship を Unihan property として保持する。`L2/19-281` は個別追加ではなく、既存 field content を置き換える 93-pair data set として提出されたため、data review と既存 value との互換性確認が重要になる。

### security confusables との接続

`kZVariant` は spoofing 専用 property ではないが、2026 年の [L2/26-127](utc-l2-26-127.md) は `kSpoofingVariant` と並んで CJK confusable pair candidate generation の根拠に使っている。したがって、この文書は [CJK Security Confusables](../topics/cjk-security-confusables.md) の前史でもある。

### review discipline

[L2/19-328](utc-l2-19-328.md) は主に `L2/19-282` の `kSpoofingVariant` data を批判したが、末尾 note で `L2/19-281` の `kZVariant` data も同じ issue を持つと述べた。CJK variant property の大量 data proposal では、PDF だけでなく text-based data、author 以外の expert review、public review の時間確保が必要になる。
