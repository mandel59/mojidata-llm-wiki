# L2/25-100

## 要約

`L2/25-100` は、Proposed Draft UTR \#59 "East Asian Spacing" の public review document である。

East Asian text に Han、Kana、Hangul と Latin letters / digits が混在するとき、可読性のために薄い visible space を自動付与する algorithm と data を定義する。

この spacing は semantic boundary としての `U+0020 SPACE` を本文に挿入するものではなく、可能なら text layout / glyph spacing の調整として扱う。

文書は `East_Asian_Spacing` property values と、grapheme cluster 単位で spacing を入れる algorithm を提案する。

## 提案内容

### property values

`East_Asian_Spacing` property は、spacing 判断のために次の値を置く。

| 値 | 意味 |
| --- | --- |
| `W` | East Asian scripts とみなす文字。Han ideographs、Kana、Hangul などが中心。 |
| `N` | `W` と隣接したとき visible spacing の対象になる文字。Latin letters や digits など。 |
| `O` | 自動 spacing の対象にしない文字。 |
| `C` | language context により `N` または `O` に解決する条件付き値。Chinese context の punctuation / symbols などを想定する。 |

### algorithm

Algorithm は grapheme cluster ごとに先頭 code point の property value を見て、`C` を language context で解決した後、隣接する `W` / `N` または `N` / `W` の間に visible space を入れる。

Vertical text では upright に表示される `N` characters を `O` として扱う。`U+0020 SPACE` は semantic boundary なので spacing 目的の代用は推奨せず、`U+2009 THIN SPACE` は spacing を明示的に入れる override、`U+200B ZERO WIDTH SPACE` は spacing を抑止する override として説明される。

### 派生 data

Draft は Script、Script_Extensions、East_Asian_Width、General_Category などを使って `East_Asian_Spacing` data を派生する規則も示す。したがって、この proposal は typography だけでなく UCD property data と algorithm text の保守にも接続する。

## 論点

### plain text と layout の境界

文書は spacing を可読性のための rendering adjustment と位置づけ、semantic boundary を表す space character と区別する。この区別は、plain text data に新しい文字を挿入するのか、layout engine が glyph spacing を調整するのかという実装境界に関わる。

### 条件付き punctuation

`C` value は Chinese language context のときだけ `N` として扱う。East Asian punctuation と Latin / digit 周辺の spacing practice は地域差があるため、property value だけでなく higher-level language context を使う点が review point になる。
