# PRI 547

## 要約

`PRI #547` は、Unicode 18.0.0 向け Proposed Update UAX \#44 Revision 37 "Unicode Character Database" の public review issue である。status は Closed、closing date は 2026-07-07。

PRI page は、UAX \#44 draft 4 が 2026-05-27 に更新され、Section 4.1 Directory Structure が UCD と Unicode release に付随する non-UCD data files の関係を明確化するために書き直されたと説明する。

UAX \#44 は UCD の layout、file format conventions、property definitions、derived properties、test files、UCD change history を説明する中核 annex であり、Unicode 18.0 beta review では data files と実装者向け documentation を確認する入口になる。

## 提案内容

### Revision 37 の主な変更

UAX \#44 Revision 37 は Unicode 18.0.0 draft 4 として公開され、modifications section は次を主な変更として挙げる。

| 区分 | 変更 |
| --- | --- |
| InCB derivation | `Indic_Syllabic_Category=Consonant_With_Stacker` を `InCB=Linker` derivation に追加。 |
| UAX \#60 data | UAX \#60 と non-Han ideographic scripts data files を説明する Section 3.8 を追加し、関連 data file / property への言及を更新。 |
| new source files | `JurchenSources.txt` と `SealSources.txt` を UCD file list に追加。 |
| directory structure | Section 4.1 を大きく書き直し、versioned data directory と UCD / non-UCD release data の関係を説明。 |

### UCD change history

Revision 37 の UCD change history は Unicode 18.0.0 の 13,047 new characters に伴う data updates を扱う。ただし draft 時点では、個別ファイルの多くが `TBD` として残っており、final UCD data での確認が必要である。

確認点としては、8 new blocks、Age value `V18_0`、new blocks / scripts の property value aliases、NamesList updates、UAX \#45 data、UAX \#14 / [UAX \#29](pri-552.md) auxiliary data などが挙がる。

## 後続決定

`PRI #547` は 2026-07-07 に closed になっている。[PRI \#548](pri-548.md) の Unicode 18.0.0 Beta review と同じ close date であり、substantive feedback の処理は UTC \#188 minutes / action items と最終 UCD / UAX \#44 published text で確認する必要がある。

[L2/26-096](utc-l2-26-096.md) は UTC \#187 の PAG report として、UAX \#31 / UAX \#44 proposed update を properties and algorithms review の一部として扱った。[L2/26-110](utc-l2-26-110.md) は UAX \#31 側の proposed update であり、identifier behavior から UCD property model に接続する。

## 論点

### UCD と non-UCD release data の境界

Revision 37 の中心は、UCD そのものと、Unicode release に同梱されるが UCD ではない data files の境界を読めるようにすることである。UAX \#38 / UAX \#45 / UAX \#60 の ideographic data、emoji data、security data、auxiliary algorithm data は、release review では同時に確認されるが、仕様上の位置づけが異なる。

このため、[Unicode 18.0 Change Sources](../topics/unicode-18-change-sources.md) では、UAX \#44 を UCD / release data の map として扱い、個別の Unihan、U-source、UAX \#60、segmentation / line break、security data は各 topic に分けて読む。

### 実装者への影響

UAX \#44 は、property aliases、property value aliases、default values、`@missing` lines、test data の位置づけを説明する。Unicode 18.0 beta で property value や auxiliary data が変わる場合、実装者は proposal 文書だけでなく UAX \#44 と beta UCD files を併読する必要がある。
