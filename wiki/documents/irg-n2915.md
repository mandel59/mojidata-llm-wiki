# IRG N2915

## 要約

`IRG N2915` は、Kim Kyongsok が `IRG N2862` の FS values に対して提出した feedback である。KR experts が N2862 を review した結果、table の各 shape で radical の有無が分からないと FS value の根拠が不明になりやすいと指摘する。

文書は、radical の有無に応じた 3 categories と Radical column の追加、さらに FS value を選ぶ原理の説明を N2862 に加えることを提案する。結論として、これらを反映した後に IRG でもう一度 review することを求めている。

## フィードバック内容

| 論点 | 内容 |
| --- | --- |
| radical categories | `[rad + comp]`、`[rad]`、`[no rad]` の 3 categories を示し、radical 以外の first stroke、radical 自体の first stroke、shape 全体の first stroke を区別する。 |
| Radical column | N2862 の table に Radical column を追加し、各 row がどの category に属するか、必要なら radical 自体を示すことを提案する。 |
| FS principle | FS values 1/2/3/4/5 の選択原理が明記されていないため、horizontal first、vertical next のような説明を追加するよう求める。 |
| Wubi concern | FS value が Wubi Input Method と関係するように見えるが、KR では Wubi がほぼ使われないため FS value の理解が難しいと述べる。 |

## 後続決定

[IRG N2862R2](irg-n2862r2.md) は、N2915 はまだ IRG meeting で討議されておらず、R2 は N2915 に基づく変更を入れていないと説明する。その後、[IRG N2909](irg-n2909.md) M66.18 は N2862R2 を accepted guideline とし、final [IRG N2951](irg-n2951.md) の準備を求めた。

## 論点

### prescriptive table と explainability

N2915 は、FS / SC table が prescriptive guideline であっても、reviewer が値を理解できるだけの説明が必要だという立場を示す。特に radical を取り除いた後の first stroke を使うのか、shape 全体の first stroke を使うのかが曖昧だと、member body 間の review が難しくなる。

### member body の入力方式差

Wubi Input Method を背景にした FS value だと読める場合、Wubi を使わない KR experts には判断原理が共有されにくい。これは IRG review infrastructure が特定地域の入力方式知識に依存して見えないよう、説明責任を持たせるべきという feedback である。
