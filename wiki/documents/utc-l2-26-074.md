# L2/26-074

## 要約

`L2/26-074` は、Unihan database に `kJapaneseNewVariant` と `kJapaneseOldVariant` という二つの provisional properties を Unicode Version 18.0 で追加する提案である。目的は、Chinese の simplified / traditional relationship を表す `kSimplifiedVariant` / `kTraditionalVariant` とは別に、日本語の 新字体 / 旧字体 関係を記録することである。

Ken Lunde は、日本語では「simplified / traditional」ではなく 新字体 / 旧字体 と呼ぶため、property names も New / Old を使うと説明する。data は Jōyō Kanji、Jinmei-yō Kanji tables、および JIS X 0208 / JIS X 0212 の関係から導いた 28 pairs に基づく。

## 提案内容

| Property | Status | Category | Introduced | Delimiter | Syntax | 意味 |
| --- | --- | --- | --- | --- | --- | --- |
| `kJapaneseNewVariant` | Provisional | Variants | 18.0 | N/A | `U\+[23]?[0-9A-F]{4}` | その ideograph の new Japanese form（新字体）。 |
| `kJapaneseOldVariant` | Provisional | Variants | 18.0 | space | `U\+[23]?[0-9A-F]{4}` | その ideograph の one or more old Japanese forms（旧字体）。 |

提案では、`kJapaneseNewVariant.txt` と `kJapaneseOldVariant.txt` を添付 data files とし、Unihan Database への property data 追加と UAX \#38 の property description 追加に必要な内容を含めるとしている。

## 後続決定

[L2/26-099](utc-l2-26-099.md) Section 21 は、`L2/26-074` に基づく `kJapaneseNewVariant` / `kJapaneseOldVariant` の追加を Unicode Version 18.0 target で受け入れるよう UTC \#187 に勧告した。[L2/26-105](utc-l2-26-105.md) の UAX \#38 Revision 40 proposed update は、Unicode 18.0.0 向けの Unihan property documentation update の入口になる。

## 論点

### Chinese variants と Japanese variants の分離

この proposal の主眼は、Japanese simplified forms を Chinese simplified / traditional properties に押し込まないことにある。U+528D 劍 の例では、中国語の simplified form U+5251 剑 と、日本語の new form U+5263 剣 が異なるため、別 property が必要になる。

### One-to-one と one-to-many

`kJapaneseNewVariant` は単一値、`kJapaneseOldVariant` は複数値を取り得る。delimiter が N/A と space に分かれるのは、new form 側と old form 側の mapping cardinality が異なるためである。
