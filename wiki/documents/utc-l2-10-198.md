# L2/10-198

## 要約

`L2/10-198` は、UTR \#45 "U-Source Ideographs" Revision 3 draft の proposed update である。文書は U-source ideographs を、UTC に提出された CJK ideograph candidates と、その提出内容・処理状態を公開記録する database として説明する。

Revision 3 は、glyph list と source data file の構造を維持しつつ、WG2 \#54 後の status changes、DYC characters の扱い、UTC source と IRG evidence requirement の関係を明確化した。特に、U-source に掲載されることは UTC が符号化を承認したことを意味せず、source field の値も current IRG evidence requirement を常に満たすとは限らない。

## データ内容

本文は U-source database を、representative glyph を並べる glyph PDF と、ideograph information を持つ UTF-8 text file の 2 files で構成されるものとして説明する。glyph は IRG work のために modern style で描かれ、original source が seal-style character であっても、seal forms 全体を符号化する仕組みを示すものではないと注意している。

Source data file は semicolon-separated の 7 fields を持つ。fields は U-source identifier、status、Unicode code point、radical-stroke index、KangXi dictionary index、IDS、source field である。identifier は `UTC` + five decimal digits で、番号は sequential に割り当てられ、skip や reuse をしないとされる。

status field は `C`、`D`、`E`、`U`、`V`、`W`、`X` を使う。`C` は Extension C、`D` / `E` は UTC Extension D / E proposal、`U` は既符号化、`V` は encoded character の variant、`W` は encoding に不適、`X` は Extension D 後の candidate を表す。`V` は IRG unification work より広い意味で使われ、component rearrangement や simplified form なども含み得る。

source field は source tag と source-specific index を組み合わせる。複数 indices は comma-separated、複数 sources は asterisk-separated とされ、URI、U-source identifier、Unihan `k*` field、dictionary / bibliographic source、WG2 document reference などが source tag として扱われる。

## 後続決定

`L2/10-198` は draft / proposed update であり、現行の UAX \#45 model は後続改訂で拡張された。[L2/25-199](utc-l2-25-199.md) では、data files が `USourceGlyphs.pdf`、`USourceRSChart.pdf`、`USourceData.txt` の 3 files になり、status values も `FutureWS`、`Variant`、`Rejected`、`NoAction`、`WS-2024` などの説明的な値へ整理されている。

## 論点

### UTC source と IRG evidence

Revision 3 の重要な整理は、UAX \#45 の source field と IRG submission evidence を同一視しない点である。文書は、U-source database に listed された sources の一部は current IRG evidence requirement を満たさない可能性があり、特に Extension D より前に集められた source sets では evidence が不足し得ると説明する。

このため U-source record は、符号化候補の存在と履歴を追跡する public record であって、IRG working set へそのまま提出できる evidence package ではない。後続の U-source proposal では、source field とは別に glyph evidence、bibliographic evidence、属性 data を確認する必要がある。

### DYC characters

文書は DYC characters について、preliminary exploration として U-source database に追加されたものであり、DYC の内容を符号化する取り組みに使う意図はないと明記する。この caveat は、dictionary-derived large source の扱いを U-source database の存在だけで正当化しないための制約として読める。
