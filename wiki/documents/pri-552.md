# PRI 552

## 要約

`PRI #552` は、UTC が Unicode 18.0.0 向けの UAX \#29 Unicode Text Segmentation Revision 48 proposed update を public review に出した PRI である。PRI page の status は Closed、closing date は 2026-07-07。

review 対象は UAX \#29 Revision 48 の draft で、draft page は Unicode 18.0.0 (draft 1)、date 2026-05-19 を示す。PRI page は draft updated を 2026-05-19 とし、Rule GB9c の変更を説明する。

変更の中心は extended grapheme cluster boundary の GB9c で、`Indic_Conjunct_Break (InCB)=Linker` と `InCB=Extend` / `InCB=Consonant` の組み合わせにおける no-break behavior を調整することである。これは Unicode 18.0 の UCD derived data と UAX \#44 の `InCB=Linker` derivation と合わせて確認する必要がある。

## 提案内容

### review window

PRI page は、UAX \#29 が Unicode 18.0 向けに更新されるため general public review and comment に出たと説明する。[PRI \#548](pri-548.md) の Unicode 18.0.0 Beta review と同じ 2026-07-07 に close しているため、UAX \#29 の最終確認は Unicode 18.0 beta package の一部として読む。

### UAX \#29 Revision 48

UAX \#29 draft の modifications section は、Revision 48 を Unicode 18.0.0 向け proposed update として記録し、Rule GB9c を `\p{InCB=Linker} \p{InCB=Extend}* × \p{InCB=Consonant}` に変更すると説明する。

draft 本文の grapheme cluster rules は、GB9c が extended grapheme clusters にだけ適用され、Indic conjunct の一部として扱う Linker / Extend / Consonant の近接で break しないことを示す。Table 1c の `conjunctCluster` regular expression には、Rule GB9c の変更を反映する更新が Unicode 18 beta に間に合っておらず、Unicode 18.0.0 final release 前に更新予定であるという note が残る。

## 後続決定

`PRI #552` は 2026-07-07 に closed になっている。UTC \#188 は [L2/26-102](utc-l2-26-102.md) の schedule 上、2026-07-30 に Unicode 18.0 content finalize を予定しているため、PRI feedback の処理結果は UTC \#188 minutes / action items、最終 UAX \#29、最終 segmentation data で確認する必要がある。

[L2/26-096](utc-l2-26-096.md) は、UTC \#187 時点で UAX \#29 GB9c と UAX \#44 の `InCB=Linker` derivation を Unicode 18.0 beta review の segmentation / derived property issue として扱っている。[PRI \#547](pri-547.md) は UAX \#44 側の UCD documentation review であり、この PRI は UAX \#29 側の algorithm text review に相当する。

## 論点

### algorithm text と UCD data の同期

GB9c は text segmentation algorithm の rule だが、実際の behavior は `Indic_Conjunct_Break` derived property と UCD data に依存する。したがって UAX \#29 の rule text、UAX \#44 の derivation documentation、beta UCD の auxiliary segmentation data を分けて確認する必要がある。

### Indic script behavior

この変更は Indic conjunct 周辺の extended grapheme cluster boundary に関わるため、[Indic Script Notation and Rendering](../topics/indic-script-notation-and-rendering.md) の valid sequence / rendering guidance と隣接する。ただし `PRI #552` は新規文字提案ではなく、既存・新規文字列をどう segment するかという algorithm / property issue である。

### beta package との関係

`PRI #552` は [PRI \#548](pri-548.md) の beta package に含まれる UAX / UTS proposed updates の一つである。Unicode 18.0 の release tracking では、UAX \#29 を UCD change history の一項目としてだけでなく、text selection、cursor movement、regular expression、segmentation tests に影響する algorithm change として読む。
