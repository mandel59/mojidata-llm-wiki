# IRG N2722

## 要約

`IRG N2722` は、Yi Bai と Eiso Chan が Unicode 16.0 で追加された JMJ horizontal extensions を IDS / glyph shape から確認し、複数の source reference / code point correspondence issues を指摘した individual contribution である。

文書は、36,000 件超の Moji Joho Kiban horizontal extensions が追加された後、JMJ references が representative glyph としてどの code point に対応するべきかに疑義が出たことを示す。

## フィードバック内容

| 対象 | 指摘 |
| --- | --- |
| JMJ-016916 / JMJ-016917 | どちらも U+72CA に horizontal extension され、JMJ-016916 が representative glyph とされるが、JMJ-016917 は U+72CA の他 source と同形、JMJ-016916 は U+3094D と同形だと指摘する。 |
| JMJ-043941 | U+25CBB に horizontal extension されているが、U+7BF9 の他 source と同形だとする。同時に J1-5258 は U+25CBB の他 source と同形だとする。 |
| JMJ-046312 / JMJ-046313 | どちらも U+26B20 に horizontal extension され、JMJ-046312 が representative glyph とされるが、JMJ-046313 は U+26B20 の他 source と同形、JMJ-046312 は U+26B07 と同形だと指摘する。 |
| JMJ-055359 / JMJ-055360 | IVDs とされるが identical shapes だと指摘する。 |

文書は、JMJ が code points の correction または respective glyphs の update を検討するよう提案している。

## 後続決定

[IRG N2859](irg-n2859.md) は、`IRG N2722` などで挙がった issues を disposition 案に落とし込み、U+72CA / U+3094D、U+26B20 / U+26B07 などについて `kIRG_JSource` value の変更、horizontal extension、no action を Japan NB に照会した。

[IRG N2870](irg-n2870.md) で Japan NB は、published encoded characters には disunification を適用しないという原則を再確認し、複数の対象について current status 維持を求めた。

## 論点

### JMJ reference と representative glyph

`IRG N2722` は、JMJ references が IVD として扱われてきた場合でも、horizontal extension 後には既存 UCS code point の representative glyph / source reference と直接衝突しうることを示す。source reference addition は data 追加に見えるが、glyph identity と code point stability の review を伴う。

### correction と disunification の境界

後続の [IRG N2859](irg-n2859.md) は source data correction のための具体案を示すが、Japan NB の [IRG N2870](irg-n2870.md) は符号化済み文字の current status 維持を優先する。この文書は、その分岐の入口として読む。
