# WG2 N5296

## 要約

`WG2 N5296` は、JMJ-source horizontal extension 後の code chart generation で、既存 J-source / JK-source characters の表示 font が HeiseiMincho / MS Mincho から IPAmjMincho へ変わって見える問題を WG2 と Japan NB に提示した feedback document である。

文書は、CJK Extension C の JK-source characters 322 字と compatibility block の J-source characters で font selection が変わり、一部に major / minor glyph changes が見えると指摘する。変更は Japan NB の明示的な request ではなく、code chart generation の J-source font handling を簡略化しようとした処理に由来すると整理している。

## フィードバック内容

| 区分 | 内容 |
| --- | --- |
| Extension C / compatibility glyphs | CJK Extension C の JK-source characters の多くが HeiseiMincho から IPAmjMincho へ、compatibility block の J-source characters が MS Mincho から IPAmjMincho へ変わり、glyph stability 上の混乱を生んだと指摘する。 |
| six glyph revert | Unicode 17.0 Alpha Review で 6 glyphs が個別に戻された結果、同じ source の characters が異なる fonts で表示される不整合が残るとする。 |
| JMJ-source characters | U+654E など 18 characters について、JMJ-source characters でありながら chart では HeiseiMincho が優先される例を示し、IPAmjMincho への変更検討を提案する。 |
| strategy options | Japan NB に対し、非 JMJ-source changes を HeiseiMincho に戻す案、J-source glyphs を IPAmjMincho に揃える案、現状維持の案を比較して明確な position を求める。 |

## 後続決定

[WG2 N5304](wg2-n5304.md) の Recommendation M72.07 は、`WG2 N5296` に記載された J-source glyph changes を Project Editor が revert するよう勧告した。wiki ではこの決定を [WG2 M72.07 J-source glyph revert recommendation](../events/wg2-m72-07-j-source-glyph-revert.md) に集約する。

## 論点

### source identity と chart font の分離

この文書の中心は、JMJ-source horizontal extension そのものではなく、既存 J-source / JK-source characters を code charts 上でどの font により代表表示するかである。source reference や owner identity を変えなくても、chart generation の font priority が変わると representative glyph が変わったように見える。

### Japan NB の position が必要な理由

文書は、font simplification の実務上の動機を認めつつ、Japan NB がどの glyph stability policy を望むのかを明確にしないと、font designers や chart readers が J-source glyph changes の意図を誤解しやすいとする。
