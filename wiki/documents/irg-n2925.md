# IRG N2925

## 要約

`IRG N2925` は、Primary / Secondary Radical assignment rules の更新提案である。WS2017 / WS2021 で、semantic radical と structural / intuitive radical のどちらを primary にするかが議論され、secondary radical が IRG review process に導入されたが、その後 secondary radicals の追加議論が会合時間と ORT manager の作業負担を増やしたことを問題にする。

文書は、lookup のための追加 radical は一般利用者には有用でも、IRG の encoding review scope から外れ得るとし、CJK & Unihan Working Group で扱う選択肢を示す。また、radical assignment は source data の品質確認・review ordering と関わるため、[IRG Indexing Rules](../topics/irg-indexing-rules.md) の中心資料になる。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| 問題 1 | Primary radical が十分 intuitive な場合にも completeness 目的で secondary radical が追加され、IRG meeting と ORT manager の負担が増えている。 |
| scope | 追加 lookup radical は dictionary data に近く、IRG ではなく CJK & Unihan Working Group で扱う余地がある。 |
| ORT action | ORT に Unihan 向け additional radicals suggestion の capability を追加する案。 |

## 後続決定

[IRG N2909](irg-n2909.md) の M66.19 は、`IRG N2925` をもとに condensed rules を含む `IRG N2952` を準備するよう求めた。`IRG N2952` は catalog には載っているが、2026-07-08 の取得時点で registry URL が 404 だったため、本文精読は未了。

## 論点

### IRG scope と Unihan maintenance の分担

`IRG N2925` の重要点は、radical assignment を「encoding review に必要な ordering / discoverability」と「一般利用者向け dictionary lookup」の二層に分けようとする点である。これは [Unihan Database Maintenance](../topics/unihan-database-maintenance.md) とも接続し得るが、この文書自体は IRG review process の負担軽減を主眼にしている。
