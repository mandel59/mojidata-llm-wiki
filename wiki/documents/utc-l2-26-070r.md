# L2/26-070R

## 要約

`L2/26-070R` は、Variation Selectors を plain-text metadata などに流用する外部仕様の動きを背景に、Variation Sequence の conformance を明確化する revised proposal である。

Variation Selector、Variation Base、Variation Sequence の formal definitions を Core Specification に追加し、standardized / emoji / ideographic variation sequence の各種定義を整理する。

Variation Selector が Variation Sequence の一部でない場合は、preceding character の glyph shape を変える request と解釈せず、missing glyph などで visible に表示すべきだとする。

Security section では、misplaced default ignorable code points が spoofing や cross-prompt injection attack に使われ得るため、可視化を検討すべきだとする。

## 提案内容

### formal definitions

文書は `Variation Selector` を `Variation_Selector` property を持つ character、`Variation Base` を decomposition / composition / combining reordering の条件を満たす graphic character、`Variation Sequence` を Variation Base followed by Variation Selector の 2-character sequence と定義する。

さらに、Assigned Variation Sequence として Standardized Variation Sequence、Emoji Variation Sequence、Ideographic Variation Sequence を区別し、意図された display effect は published lists / IVD registration に限ると整理する。

### misplaced VS と security

CR の後、文字列冒頭、あるいは `A<VS2><VS1>` の最後の VS1 のように、Variation Sequence を構成しない VS は misplaced とされる。提案は、こうした default ignorable code points を silently ignored にするだけでなく、通常表示でも visible にする選択肢を示す。

## 後続決定

[L2/26-096](utc-l2-26-096.md) は、`L2/26-070R` について revised document が誤った DICP definition を取り下げたため UTC \#187 では action せず、後続改訂で扱うと整理した。したがって、この proposal は conformance issue の入力文書であり、UTC \#187 時点の最終決定ではない。

## 論点

### Default_Ignorable_Code_Point の正当な文脈

Default ignorable は、仕様で定義された目的に使われるときに fallback として ignore できる property であり、任意の hidden metadata channel ではない。文書はこの distinction を security guidance と conformance definition に接続する。

### 複数 VS sequence の扱い

複数 VS が連続する sequence を formally non-conformant にする問題は、IVS、emoji variation sequence、standardized variation sequence の既存 behavior とも関わる。実装は unsupported assigned / unassigned VSeq の fallback と、misplaced VS の可視化を分けて扱う必要がある。
