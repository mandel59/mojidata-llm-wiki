# WG2 N5310R

## 要約

`WG2 N5310R` は、Jinping dialect of Dai language を記録する Tai Xaau script を UCS に追加する revised proposal である。China National Body が提出し、`WG2 N5310` への revision として、Tai Viet との関係を大きく書き直している。

提案は 76 Tai Xaau characters を新 script として扱う。内訳は 44 consonant letters、7 vowel letters、11 vowel marks、2 tone marks、12 symbols / punctuation で、summary form では contemporary category、common use、complete proposal とされる。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| script | Tai Xaau。Jinping Dai writing system とも呼ばれ、Yunnan Province の Jinping Miao, Yao, and Dai Autonomous County などで使われる。 |
| repertoire | 76 characters。44 consonant letters、18 vowel letters / marks、2 tone marks、12 symbols / punctuation。 |
| encoding model | pre-base vowels があるが、all consonants が base character として同じに振る舞うため、logical order ではなく visual order を提案する。 |
| Tai Viet との関係 | font style、glyph shape、user identity の差を根拠に、Tai Viet から disunify して独立 encoding とすることを提案する。 |
| 後続 | WG2 \#72 M72.15 は proposal の progress を note し、authors に継続作業を促した。`WG2 N5368` が補足資料として提出された。 |

## 後続決定

[WG2 N5304](wg2-n5304.md) の M72.15 は、Dai Xaau proposal の progress を note し、proposal authors に継続作業を促した。これは acceptance ではなく、追加作業を求める progress note として扱う。

[WG2 N5368](wg2-n5368.md) は、`WG2 N5310R` の補足として、Dai Xaau / White Dai と Dai Viet / Black Tai の form、sound、meaning の差、追加文字、numerical abbreviations を説明した。

## 論点

### Tai Viet との disunification

文書の中心論点は、Tai Xaau を既存 Tai Viet script の glyph / localization variation と見るか、独立 script として disunify するかである。`WG2 N5310R` は、available Tai Viet fonts と Tai Xaau fonts の style が重ならないこと、Tai Xaau community の glyph comparison、Tai Viet / Tai Xaau user identity の差を根拠に disunification を主張する。

### visual order encoding model

Tai Xaau は Tai Tham や New Tai Lue と同様に pre-base vowel letters を持つが、subjoined consonants を持たず、consonants は base units として同じに振る舞う。文書は、vowel killer のような control character を導入すると混乱するため、visual order model を採る方が implementation を簡単にすると説明する。

### tone marks と mark placement

tone marks は syllable 全体を修飾し、literature では複数の grapheme cluster 上に置かれ得る。文書は、tone mark の位置違いによる sequences を equivalent として扱う必要があるとし、入力・検索・normalization の設計課題を残す。
