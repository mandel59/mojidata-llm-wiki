# L2/25-267

## 要約

`L2/25-267` は、UTR \#25 Unicode Support for Mathematics Revision 16 draft 2 の L2 登録文書である。draft は Base Unicode Version を Unicode 17.0.0、Date を 2025-10-02 とし、[PRI \#533](pri-533.md) の review 対象になっている。

文書は Unicode の mathematical character repertoire、mathematical alphanumeric symbols、math character properties、implementation guidelines、math data files、security considerations を扱う。Revision 16 では、主に PDF から HTML への移行、accessibility / navigation / references の改善、math data files の更新が行われている。

内容面では、Unicode 9 support から Unicode 17 support へ更新する本格的な content revisions は今後の作業として説明されているため、この文書は最終的な modernized UTR \#25 ではなく、その土台になる draft として読む。

## 提案内容

### Revision 16 draft 2

Revision 16 の modifications section は、report の HTML conversion、W3C HTML-MathML Entity definitions に合わせた data files update、Unicode General Category の listed data への追加、Section 3.1.3 Multiple Use、Section 3.2.7 Combining Marks の single text runs paragraph、references / links / character listing consistency の整理を記録する。

### UTR \#25 の scope

UTR \#25 は、Unicode Standard 本体の conformance rule ではなく、数学文字 support の informative technical report である。数式で使われる plain text characters、MathML / OpenMath / TeX / UnicodeMath、programming language での use、math character properties、math data files を横断して扱う。

### data files

文書は、mathematical notation における primary usage で characters を分類する `MathClass-16.txt` と、SGML / MathML entity sets などへの mapping を持つ `MathClassEx-16.txt` を参照する。`Math` property を持つ characters は classification に含まれるが、formal default `Math_Class` assignments はないと説明する。

## 後続決定

`L2/25-267` は `PRI #533` の review 対象であり、2026-07-10 時点では PRI page / catalog が Open のままである。UTC \#188 の feedback handling、最終 UTR \#25 Revision 16、math data files の publication を確認する必要がある。

## 論点

### HTML migration と content update

Revision 16 は structural cleanup が中心である。Unicode 17 までの repertoire / property / examples / table updates は今後の significant content revisions として予定されているため、現時点で draft 2 の本文を最新 complete guidance と見なさない。

### mathematical text と security

Mathematical alphanumeric symbols は数学的 semantics を保持するために plain text character として使われる一方、styled text と confusable になる。文書は、domain names など security sensitive environments ではこれらが除外されることを UTR \#36 へ接続して説明する。
