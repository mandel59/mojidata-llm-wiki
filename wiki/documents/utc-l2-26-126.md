# L2/26-126

## 要約

`L2/26-126` は ICU4X と ECMA TC39 の liaison update である。ICU4X 側は CLDR / TZDB updates、calendar behavior、UCD property support、segmentation、data loading を報告し、TC39 側は Intl / Temporal proposals の進行を報告する。

UTC \#187 の文脈では、この report は Unicode 18.0 の標準文書そのものではなく、Unicode data と algorithms が実装 ecosystem にどう流れ込むかを示す資料である。特に ICU4X の UCD direct loading と properties support は、UCD distribution と implementation の接点として読む必要がある。

## 報告内容

| 区分 | 要点 |
| --- | --- |
| ICU4X data | CLDR 48.2 / TZDB 2026a 対応、日本暦 pre-Meiji eras の deprecation、Hijri Reingold simulation の default calendars からの移動を報告する。 |
| ICU4X APIs | date arithmetic APIs、typed Month enum、`until()` optimization、Kotlin bindings、experimental ML segmentation などを報告する。 |
| Unicode properties | Numeric_Type、Joining_Group、Indic_Conjunct_Break の support を報告する。 |
| UCD consumption | `--ucd-root` / `--ucd-tag` / `--unihan-root` の flag refactoring と、UCD data を直接読む方向を報告する。 |
| TC39 Intl | Intl Locale Info、Temporal、Intl Era and MonthCode が Stage 4 ready for ship とされる。 |
| TC39 proposals | Intl Keep Trailing Zeros、Intl Unit Protocol、String toTitleCase、Amount、Intl Energy Units などの進行を報告する。 |

## 後続決定

UTC \#187 minutes [L2/26-093](utc-l2-26-093.md) は liaison reports を会合記録に含める。TC39 proposals 自体の仕様決定は ECMA TC39 側の process に属するため、この page では UTC への liaison update として扱う。

## 論点

### Unicode data の利用側からの feedback

ICU4X は UCD / Unihan / segmentation / collation などを実装に取り込む側であり、release artifact の構造変更に直接影響を受ける。Unicode 18.0 の release coordination を見る際には、RMG / Charts / EDC だけでなく実装側 liaison も確認する必要がある。

### TC39 Intl と Unicode algorithms

Temporal、Intl Era and MonthCode、String toTitleCase などは、Unicode / CLDR の data と algorithm naming に依存する。Liaison report は、UTC が直接決める文字追加とは別に、Unicode ecosystem の API surface がどこで議論されているかを示す。
