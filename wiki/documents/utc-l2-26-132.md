# L2/26-132

## 要約

`L2/26-132` は KORE SEBELI / Wakara script を Unicode に符号化する proposal である。UTC register では 2026-05-14 に登録されているが、抽出テキスト上は April 2026 version 12 として [L2/26-050R](utc-l2-26-050r.md) と同じ内容を持つ。

表紙・目次・proposal summary form は 93 characters とする。一方、本文の proposed characters section は 91 characters と数え、38 capital letters、38 lowercase letters、3 punctuation marks、2 other mathematical signs、10 digits / numerals を列挙する。この 93 / 91 count の不一致は UTC review で確認すべき点である。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| script scope | Soso language 用の contemporary Guinean script。Koré Sébèli / Wakara とも呼ばれる。 |
| version | April 2026 version 12。UTC register entry は 2026-05-14。 |
| repertoire | 93 characters とする箇所と、91 characters とする本文 section が併存する。38 capital、38 lowercase、10 digits、punctuation / mathematical / other signs が中心。 |
| direction | horizontal left-to-right を default とし、vertical direction は elderly / traditional use、divinatory / talismanic context、sample として説明する。 |
| atomic model | dots は letter shape の一部で、base letter + dots を single atomic character とし、canonical decomposition しないと説明する。 |
| case model | uppercase / lowercase の使い方は Latin alphabet と同じだと説明する。 |
| community evidence | teachers、students、teaching places、handwritten samples、syllabary、font use、publication / school plans を含む。 |

## 後続決定

2026-07-09 時点では、UTC \#188 agenda / minutes は catalog に未掲載であり、この proposal への UTC 決定は未確認である。後続の disposition が出たら、[KORE SEBELI Script](../topics/kore-sebeli-script.md) と [Script Encoding Pipeline](../topics/script-encoding-pipeline.md) 側で採否・保留・修正要求を追う。

## 論点

### 93 / 91 count

同じ version 12 の中で 93 characters と 91 characters が併存する。proposal summary form、code chart、property table、names list のどれを採用対象とするかは、UTC / SEW disposition の確認が必要である。

### atomic dotted letters

2026 年版では、dots を combining marks ではなく letter shape の一部とし、dotted vowel / consonant letters を atomic characters として扱う。2020-2022 年版の combining diacritics / vowel sequences との関係が review point になる。

### contemporary invented script の review

KORE SEBELI は contemporary script として提出されているため、community use、literacy / education evidence、orthographic stability、case model、digits / punctuation を同一 script block に含める妥当性が重要になる。
