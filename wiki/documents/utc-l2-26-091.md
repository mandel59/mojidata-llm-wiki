# L2/26-091

## 要約

`L2/26-091` は UCD `StandardizedVariants.txt` にある Mongolian character variants を deprecate し、関連する Core Specification text を後続 revision で更新する proposal である。現行の standardized variants が current practice と China national standards に同期していないことを背景にする。

文書は、Mongolian variants は free variants ではなく orthographically required / semantically significant であり、FVS は variant decider ではなく switcher として使われてきたと整理する。さらに、font implementation timing、writing-system-specific requirements、UCD data model の不足を deprecation 理由に挙げる。

## 提案内容

| 区分 | 内容 |
| --- | --- |
| request | Mongolian standardized variants in `StandardizedVariants.txt` を deprecate する。 |
| targets | Core Specification と UCD text / data の後続 revision。 |
| rationale | orthographic significance、FVS function mismatch、font implementation expectation、writing-system-specific variant requirements。 |
| feasibility | `StandardizedVariants.txt` に commented-out variant precedent があるため、UTC resolution による comment-out が可能とする。 |

## 後続決定

[L2/26-100](utc-l2-26-100.md) は、Mongolian standardized variants について既存 standardized variation sequences を削除せず、backwards compatibility を保つ revised proposal を求める方向として扱う。2026-07-09 時点の local catalog では、この deprecation proposal への採択決定は未確認である。

## 論点

### standardized variant と orthographic requirement

提案の中心は、Mongolian variants が optional / free variation ではなく writing-system-specific rules と semantic distinction に関わるため、`StandardizedVariants.txt` の standardized variant model に合わないという点である。既存 encoded text との互換性を保ちながら UCD data と Core Specification guidance をどう整理するかが問題になる。
